This page concentrates all information about KDevelop 1.x release series

Development for KDE 1.x has stopped with on 11. December 2000 with the KDevelop 1.3 release, and no further KDevelop 1.x releases are planed for KDE 1.x.
KDevelop 1.4 is for KDE 2.x and Qt 2.x. but please use KDevelop 2.1.5 instead if you need a KDevelop for KDE 2.x.
Currently only four wishes remain in KDevelop 3.0 (for KDE 3.0, Qt 3.0) from the many bugs and wishes of KDevelop 1.3.

Compiling version 1.3 from CVS repository

To get a self-compiled KDevelop 1.3 from cvs sources, checkout CVS module "kdevelop" at :pserver:anonymous@anoncvs.kde.org:/home/kde using the bugfix CVS branch KDEVELOP_1_0

command line if you are running KDE 1.x:
       $ mkdir kdesrc
       $ cd kdesrc
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDEVELOP_1_0 kdevelop
       $ cd kdevelop

In order to compile and install KDevelop (CVS-version) on your system,
type the commands from below in the base directory of the KDevelop distribution:

a release-compiled version: 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

a debug-compiled version: 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)

Known bugs

Because development on this release series has stopped, there are many known but unfixed bugs. The good news is that the bugs are all fixed in KDevelop 3.0. So please consider using version 3.0 instead.
