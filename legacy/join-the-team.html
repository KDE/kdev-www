There's no reason to grumble about bugs and missing things. KDevelop can only be as good as you help to improve it.
So we need you! Our team always looks for good Qt or KDE programmers.


So what is to do to become a team member?:
  • have an idea what is annoying, bad or missing in KDevelop and have the will to implement that.
  • get the sources of kdevelop from SVN
  • compile yourself everything as debug version to be able to backtrace crashes
  • study the kdevelop source code, debug KDevelop itself and watch the running backtraces to understand the mechanisms; recognize the important and unimportant C++-classes in the project
  • patch the current sources
  • create a patch file (*.diff) against the current cvs version
  • Send it to kdevelop-devel at kdevelop.org
  • get response, flaming and love
  • write more such cool patches and bother the mailing-list again with it
  • learn the SVN commit policies
  • get an own kdevelop-cvs account, and maybe a yourname at kdevelop.org address to be able to commit your stuff by yourself
  • ...
  • *years of becoming a long staying team member* (seeing the people, patches and versions come and go)
  • ...
  • become a KDevelop guru
  • ...
  • become rich due thousands of fan gifts and donated stuff... ;-)
  • ...
  • KDevelop-20.5.134 release
  • ...
  • The End.
