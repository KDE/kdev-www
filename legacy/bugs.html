To enter a new bug report or feature wish, visit the bug report wizard of our bugzilla database. You should not report compilation errors to the bug database. For compilation errors use the mailing list instead.

For the a full report on KDevelop's current bugs and wishes query the bug database. The reports are seperated into components, so you can also get a Report Count by Component and Severity.

To have a nice detailed view of the bugs, enable cookies on your browser then go to the bugzilla column configuration page and select:
  • changeddate
  • severity
  • priority
  • reporter
  • status
  • component
  • version
  • votes
  • Summary (first 60 characters)

That's all, from now on all your bug queries will have all the details you need for KDevelop.
Happy bug hunting! :)
