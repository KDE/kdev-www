本页主要提供KDevelop 1.x 系列的所有主要信息

KDE 1.x的开发已随着KDevelop 1.3的发布，于2000年12月11日中止，我们不打算再继续在KDE 1.x平台上继续KDevelop 1.x系列的开发。
KDevelop 1.4是为KDE 2.x和Qt 2.x发布的。但是如果您希望在KDE 2.x上运行KDevelop，请使用KDevelop 2.1.5。
目前来自KDevelop 1.3的需求仅剩四个没有在KDevelop 3.0 上(用于KDE 3.0, Qt 3.0)实现。

从CVS编译1.3版本

要从cvs获得可以自行编译的KDevelop 1.3，从 :pserver:anonymous@anoncvs.kde.org:/home/kde 获取CVS模块"kdevelop"，选择缺陷修复的CVS分支 KDEVELOP_1_0

如果您运行KDE 1.x，命令行为:
       $ mkdir kdesrc
       $ cd kdesrc
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDEVELOP_1_0 kdevelop
       $ cd kdevelop

要在您的系统编译和安装KDevelop (CVS版本)，
在KDevelop发行的基目录下输入下列命令：

一个发行编译版本： 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

一个调试编译版本： 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)
