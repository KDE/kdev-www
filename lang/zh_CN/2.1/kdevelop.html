本页主要提供KDevelop 2.x 系列的所有主要信息

开发已于2002年11月19日中止，最后一个版本是随KDE 3.0发布的KDevelop2.1.4，我们不打算再继续2.x系列的开发。 KDevelop 2.1.5作为KDE 3.1用户的升级是一个例外，发布于2003年1月28日。开发转移到新的KDevelop 3.0的代码基础上。 80%以上的报告的KDevlop 2.x的缺陷 KDevelop 3.0得到解决。

如何安装 KDevelop 2.1.5

详细的指南可以在如何在Suse 8.2下安装KDevelop 2.1.5 (28KB .txt)找到。 其中也包括如何安装C/C++参考。

从CVS编译2.1.5版本

要从cvs获得可以自行编译的KDevelop 2.1.5，从 :pserver:anonymous@anoncvs.kde.org:/home/kde 获取CVS模块"kdevelop"，选择缺陷修复的CVS分支 KDE_2_2_BRANCH
如果您运行KDE 3的话，您还需要 从cvs.kde.org获取kde-common，来获得admin目录下的内容。
请注意不同的KDE 3的版本需要不同的cvs分支 ：
  • KDE-3.0.x 需要 KDE_3_0_BRANCH,
  • KDE-3.1.x 需要 KDE_3_1_BRANCH,
  • 只有cvs HEAD不需要kde-common的cvs标签,
这也是为什么我们不使用 '-r tagName_seeAbove'选项的原因。

如果您运行KDE 3，命令行为:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r tagName_seeAbove kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

如果您运行KDE 2.2，命令行为:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

要在您的系统编译和安装KDevelop (CVS版本)，
在KDevelop发行的基目录下输入下列命令：

一个发行编译版本： 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

一个调试编译版本： 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)
注意：对于KDE 3您至少需要Qt > 3.0.3 (最好是KDE的qt版本), 对于KDE 2.2.x您需要Qt 2.3.x
避免同时编译两个Qt版本，Qt2和Qt3!否则，KDevelop会启动 失败，产生一个莫名其妙的崩溃报告...

插件和补丁

由于多种原因(主要是稳定性), 一些大的补丁没有应用到现在的版本上。 但是它们提供了很大的改善，您可以在这里下载它们。
请记住这些是源代码补丁，所以您必须下载兼容的源代码包，打补丁并重新编译。

消息处理,断点和监视窗口(对KDE/QT-3.0.x很重要) +
