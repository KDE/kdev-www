它包含了所有 KDevelop 3.0.x 的特色 以及:
  • Code Snippets
    • SnippetPart 增加一个工具视图，缺省停靠在右边
    • 可以通过弹出菜单增加，编辑和删除snippets
    • 双击一个snippet会将它插入当前视图的光标所在位置
    • 工具条提示可以显示snippet的上下文
    • Snippets保存在用户目录下，所以每个用户都可以有自己的snippets
    • Snippets可以包含类似 $VARNAME$的变量。使用这样的 snippet，会提醒用户替换这些变量的值
  • Doxygen 集成 (API文档生成器)
    • 支持doxygen支持的所有编程语言(除了IDL 和D)
    • 可以同步预览doxygen对您正在编辑的文件生成的文档
    • 配置 doxygen
    • 运行 doxygen
    • 查看 doxygen 生成的文档 (html man 或 pdf 格式)
    • 可以用 doxygen生成的 XML作为代码自动补全的提示
  • 基于插件的文档查看器
    • 统一的文档浏览界面
    • TOC, 索引和全文搜索(采用htdig)。
    • 自动检测，包含Qt, KDE库文档和DevHelp文档。
    • 支持自定义的文档集合(KDevelopTOC 格式)。
    • 可以打开本地、远程，甚至压缩文件(采用Konqueror支持的协议 - http, ftp, sftp, tar, zip和其他)。
    • 支持TXT, HTML, DevHelp, CHM, PDF, Djvu 和 PDB 文件格式。
    • 可以使用任何您的KDE系统所支持的文档
  • 灵活的应用程序向导(项目模板)
    • 应用程序向导可以生成完整的，立即可用的样板程序。
    • 各种支持的语言和编译系统都有相应的模板。
    • 可以使用发行包以外的自定义模版。
    • 包含所有的 KDevelop 3.0 模板以及GTK, GNOM E和 Bonobo应用程序模板
    • 支持其他只有 Ian 知道的东西 :)
  • 面向快速开发(RAD，Rapid Application Development)
    • 新的kdevdesigner应用通过集成QTdesigner，可以在KDE的框架内无缝的集成基于QT的GUI开发。
    • 新的kdevassistant应用允许 QT风格的 文档帮助。
    • 这两个程序可以作为独立的应用或者通过KDevelop来调用， 这使得 一个开发项目可以满足您所有的开发需求。
