Esta página concentra toda la información de las series de liberaciones de KDevelop 1.x

El desarrollo para KDE 1.x está parado desde el 11 Diciembre 2000 con la revisión de KDevelop 1.3, y no hay planeadas futuras revisiones de KDevelop 1.x.
KDevelop 1.4 es para KDE 2.x y Qt 2.x. pero por favor mejor utilice KDevelop 2.1.5 si necesita un KDevelop para KDE 2.x.
Actualmente solo quedan cuatro peticiones en KDevelop 3.0 (para KDE 3.0, Qt 3.0) desde algunos de los bugs y peticiones de KDevelop 1.3.

Versión Compilable 1.3 del repositorio de CVS

Para obtener un KDevelop 1.3 auto-compilable desde los fuentes de cvs, CVS module "kdevelop" at :pserver:anonymous@anoncvs.kde.org:/home/kde usando la rama correguida de CVS KDEVELOP_1_0

línea de comandos si esta corriendo KDE 1.x:
       $ mkdir kdesrc
       $ cd kdesrc
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDEVELOP_1_0 kdevelop
       $ cd kdevelop

La orden para compilar e instalar KDevelop (versión CVS) en su sistema,
escriba los comandos de abajo en el directorio base de la distribución de KDevelop:

una versión compilada para su liberación: 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (como root)

una versión compilada para depurar: 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (como root)

Bugs conocidos

Debido a que el desarrollo de esta serie de revisiones está parada, hay muchos bugs conocidos y no corregidos. Las buenas noticias son que todos los bugs están corregidos en KDevelop 3.0. Así que considere usar la versión 3.0.
