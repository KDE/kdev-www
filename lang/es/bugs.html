Para entrar un nuevo informe de bug o una petición de funcionalidad, visite la base de datos asistente de informe de bug de nuestro bugzilla. No debe informar de errores de compilación en la base de datos de bugs. Para los errores de compilación use la lista de correo.

Para ver un informe completo de los bugs actuales y las peticiones de KDevelop consulte la base de datos de bug. El informe está dividido por componentes, por lo tanto también puedes obtener un Infome Numérico por Componente y Gravedad.

Para tener una lista más detallada de los bugs, active las cookies en su navegador y vaya a la página de configuración de columna de bugzilla y seleccione:
  • changeddate (fecha de cambio)
  • severity (gravedad)
  • priority (prioridad)
  • reporter (informante)
  • status (estado)
  • component (componente)
  • version (versión)
  • votes (votos)
  • Summary (first 60 characters) Resumen (primos 60 caracteres)

Esto es todo, desde ahora todas las consultas que haga tendrán todos los detalles que necesites para KDevelop.
