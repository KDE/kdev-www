Antes de empezar a compilar su reciente versión de KDevelop, asegúrese que tienes instalado los siguientes programas. Recuerde que puedes tener errores extraños, si usa versiones antiguas.
Normalmente puede ver que versón del software está instalado en su computador ejecutando el programa con el argumento "--version". Por ejemplo para buscar que autoconf tiene instalado en su máquina haga: 
   autoconf --version

Programa/Biblioteca Fuente
KDE >= 3.0.2 www.kde.org
Qt >= 3.0.5 www.trolltech.com/products/qt
g++ >= 2.95.3 (o compatible) gcc.gnu.org
GNU make www.gnu.org/software/make
perl >= 5.004 www.perl.com
autoconf >= 2.52 www.gnu.org/software/autoconf
automake >= 1.6 www.gnu.org/software/automake
flex >= 2.5.4 www.gnu.org/software/flex
Berkley DB >= 3.0 and <= 4.1 www.sleepycat.com


Puede evitar la necesidad de tener autoconf >= 2.52 y automake >= 1.6 haciendo: 
      $  cd KDevelopSourceDirectory
      $  rm -Rf admin
      $  ln -s $KDEDIR/share/apps/kdelibs/admin admin
Si esto no funciona, entonces necesitará realmente actualizar autoconf y automake. Estas herramientas son opcionales, esto significa que KDevelop funciona sin ellas. Pero carecerá de algunas funcionalidades como generar documentación o realizar búsquedas sobre varios ficheros. Así que probablemente deba comprobar si en su sistema están estos programas.

Por favor dese cuenta que para usar lenguajes de programación soportados como ada, pascal, java, etc. necesitará el correspondiente compilador del lenguaje. KDevelop no limita el compilador a usar pero ofrece diálogos de opciones de compilación para sólo algunos de ellos. Actualmente se disponen de los siguientes plugins de compilación gcc, g++, f77, pghpf (Portland Group High Performance Fortran Compiler), pgf77 (Portland Group Fortran 77 Compiler), dcc (Borland Delphi Compiler) y fpc (Free Pascal Compiler).
