• Arquitectura basada en Plugins
    • Se puede añadir, reemplazar y eliminar funcionalidad sin alterar el núcleo del código fuente.
    • La licencia de bibliotecas (LGPL) permite crear plugins IDE comerciales y libres.
    • Soporta los perfiles para determinar el conjunto de plugins que serán cargados. Esto es especialmente útil para entornos restringidos o instalaciones personalizadas.
  • Gestión del proyecto
    • Soporte nativo y no intrusivo de automake y qmake. Se permiten las modificaciones manuales y se reconocen por el IDE, también se puede importados fácilmente proyectos y subproyectos ya existentes.
    • Capacidad de usar makefiles personalizados para construir el proyecto.
    • Soporte para ant - herramienta de construcción para proyectos java.
    • Soporte extra para tipos de proyectos como pascal, haskell, ada y proyectos de scripting.
    • Un entorno de trabajo para añadir soporte para cualquier sistema de construcción mediante la creación de plugins de sistema de construcción para el gestor de proyecto genérico.
    • Nuevo asistente de "Nuevo fichero" para gestionar las plantillas de ficheros y crear nuevos ficheros en el proyecto a partir de plantillas.
    • Plugins de compilación para definir las opciones de compilación en útiles diálogos.
    • Configuraciones personalizadas con la capacidad de especificar diferentes compiladores, flags de compilación, directorios de construcción, arquitectura destino, etc.
    • facilidades para documentar el proyecto como el soporte de doxygen para la creación automática de la documentación del API (en formato html, tex o man) para un proyecto con referencias cruzadas a las bibliotecas usadas.
    • Soporte de internacionalización, permite a los traductores añadir fácilmente idiomas al proyecto, incluyendo soporte para KBabel.
    • Soporte para proyectos Qt/Embedded(Embebido) (para las plataformas Zaurus y iPAQ).
  • Soporte de Lenguajes
    • 15 lenguajes soportados incluyendo Ada, C, C++, Objective-C (a través del soporte C), SQL, Fortran, Haskell, Java, PHP, Pascal, Perl, Python, Ruby, Bash, XUL (de forma no oficial).
    • Informador de problema que informa de varios "problemas" con el código fuente para cada tipo. Entre los Problemas se incluye los errores de sintaxis, de TODO y de FIXME.
    • Navegador de clase para visualizar la jerarquía de clases y otros símbolos del proyecto.
    • Parsecs de lenguajes para alimentar al navegador de clase y al informador de problemas.
    • Asistente de clase, método y atributo para lenguajes orientado a objeto.
    • Completación automática de código e indicación de código para variables tipo clase, métodos, argumentos de funciones y más.
    • Formateo de fuentes, resaltado de sintaxis y plegado de código.
    • funcionalidades específicas de C++, como
      • "make member" - crear la definición de un miembro en el fichero fuente a partir de la declaración del método en el fichero de cabecera (header);
      • "extract interface"- extracción del interfaz (declaraciones de métodos) y creación de la clase base abstracta con esa interfaz;
      • "subclassing" - creación automática de subclase e implementación del slot para los formularios de Qt Designer.
  • Depurador Integrado
    • interfaces para los depuradores GDB y Java.
    • Integración con GDB de consola.
    • Línea, dirección, método de puntos de ruptura y "watchpoints".
    • Puntos de ruptura condicionales.
    • Desensamblador y visor del marco de pila.
    • Árbol de vistas de variables.
    • Visores de memoria, registros y bibliotecas.
    • Soporte de depurador de intérprete de comandos (libtool).
    • Depuración remota.
  • Edición de Código
    • Cualquier editor para el cual esté implementado el interfaz de KDE KTextEditor puede usarse con KDevelop.
    • Actualmente se disponen de los editores Kate, QEditor (basado en el editor de Qt Designer) y KVim.
    • Marcadores globales del proyecto que se almacenan en el fichero de proyecto.
    • Visor de diferencias entre la el texto editado actualmente y el texto en el disco.
    • Visor de diferencias para los ficheros de parches.
    • Integración de Grep - buscador de expresiones regulares en el proyecto.
    • Herramienta de búsqueda y sustitución global del proyecto.
  • Sistema de Control de Versión
    • Un API común para incluir el soporte de SCS para KDevelop.
    • Sistemas VCS soportados: CVS, Perforce, Subversion y Clearcase.
  • Asistente de Aplicación
    • El asistente de aplicación genera aplicaciones ejemplo completas y listas para funcionar.
    • Plantillas para cada uno de los lenguajes soportados y sistema de construcción existente.
    • Una rama de KDE relacionada con las plantillas del proyecto (aplicaciones KPart, aplicaciones simples para KDE, módulos KControl, Kicker (panel) applets, KOffice parts, KIOSlaves, Konqueror, Noatun y plugins de Kate y estilos de escritorio).
  • Visor de Documentación
    • Un interfaz unificado para la navegación de la documentación.
    • TOC, capacidades de indexar y búsqueda de texto completo (con htdig).
    • Autodetección e inclusión de la documentación de Qt, KDE libs y DevHelp.
    • Soporte para colecciones de documentación personalizada (formato KDevelopTOC).
    • Capacidad para abrir documentación local, remota e incluso comprimida (usando cualquier protocolo soportado por Konqueror - http, ftp, sftp, tar, zip, etc.).
    • Capacidad para abrir varios formatos de documentación (html, text, ps, pdf, chm, etc.).
  • Otras Funcionalidades
    • El interfaz de usuario es completamente configurable, y permite cambiar al vuelo entre los modos clásicos MDI como toplevel, childframe y el nuevo modo IDEAl con vistas de herramienta deslizables.
    • consola embebida.
    • Facilidades de selector de fichero, árbol de ficheros y grupos de ficheros que hace innecesario un administrador de ficheros externo.
    • La inclusión de cualquier otro programa que necesite para desarrollar añadiéndolo en el menú "Herramientas" acorde con su necesidades individuales.
    • Comprobador de expresiones regulares.
    • Integración con Valgrind para encontrar perdidas de memoria en los programas.
    • Distribución de proyecto y herramientas de empaquetado para automatizar la creación de paquetes fuentes y binarios.
    • Integración con CTags.
    • Explorador de Part para consultar la disponibilidad de servicios KDE.
