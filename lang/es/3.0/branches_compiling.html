Obten los fuentes para el KDE_3_2_BRANCH como se explica abajo.
Antes de continuar por favor comprueba si su computadora tiene instalado el software requerido.
Inicie el sistema build: 
       $ cd kdevelop
       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs

Después descargar y compilar con exito una de las ramas de CVS de mas arriba, querrá mantenerlo actualizado obteniendo todos los parches. Para evitar escribir en todas sentencias que que defina las variables de entorno necesarias cada vez que actuaize, debe definir las en su fichero .bashrc o .cshrc.
Si usa la shell bash añada las siguientes líneas:
export KDEDIR=/where/your/kde3/is
export QTDIR=/where/your/qt3/is
export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH
Si usa el intérprete de comandos tcsh añada las siguientes líneas:
setenv KDEDIR /where/your/kde3/is
setenv QTDIR /where/your/qt3/is
setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
setenv PATH $QTDIR/bin:$KDEDIR/bin:$PATH

Nota: por razones obvias debe editar las primeras dos líneas para adecuarlas a su configuración! :)


Después vaya al directorio donde ha puesto los ficheros kdevelop desde CVS y proceselo con el comando de actualización:
 $ cd where_kde3src_is/kdevelop
 $ cvs update -dP

El siguiente paso depende de la salida del comando anterior:
Si obtuvo algo parecida a:
           U /some_directory_path/Makefile.am
Debe hacer:
 $ make -f Makefile.cvs
y después _TODOS_ los pasos siguientes.

Si obtiene algo parecido a:
           P /some_directory_path/Makefile.am
Debe hacer:
 $ ./configure --enable-debug=full --prefix=$KDEDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
y después _TODOS_ los pasos siguientes.

  $ make
  $ make install (como root)
Si por alguna razón el comando make falla puede forzarlo para que ignore los errores añadiento la opción -k.Dependiendo de la gravedad del error podría acabar con una versión de kdevelop correcta o incorrecta. ¡¡¡Así que utilice "make-k install" bajo su propio riesgo!!!. De todos modos estos errores normalmente desaparecen después de un par de horas así que podría querer intentar una actualización otra vez antes más tarde antes de peligroso :) comando make install.
 $ make -k
 $ make -k install (como root)
¡Esto es todo! Ahora tienes una versión superior de KDevelop. Para comenzar haga:
 $ kdevelop
Todo lo que tiene que hacer para mantenerse al día es repetir estos pasos cuando alguien en la kdevelop-devel lista de correo corrige un bug.
Por lo tanto le recomendamos encarecidamente que se una a esta lista de correo.


Debe informar de los bugs que encuentre con el formulario de envío de bug. No debe informar de errores de compilación en la base de datos de bug. en lugar de eso, para los errores de compilación use la lista de correo.

Puede echar un vistazo de los bugs en la base de datos de bug.

Siga las instrucciones normales, pero use gmake en vez de make y después 
       $ gmake -f Makefile.cvs
haga: 
       $ export CXXFLAGS="-L/usr/X11R6/lib -I/usr/local/include/db41 -I/usr/local/include -L/usr/local/lib -I/usr/X11R6/include"
Note que necesita la berkeley db 4.1 instalada, si tiene otra versión, necesita ajustar la ruta.
Vambie la línea "ac_cv_bdb=no" por "ac_cv_bdb=yes" en el fichero configure por que de otro modo, configure no puede encontrar las bibliotecas db.
Para ver que versión tiene instalada: 
       $ pkg_info | grep db41
       db41-4.1.25_1       The Berkeley DB package, revision 4.1
Los chicos de FreeBSD han renombrado libdb.so por libdbxx.so. Entonces sustituya "-ldb" by "-ldb41" en lib/catalog/Makefile.am
Después puede continuar el proceso normal: 
       $ ./configure --prefix=/blah --enable-debug=full
