Para nos comunicar bus e desejos visite a página de comunicação de bugs da nossa base de dados bugzilla. Voçe não deve comunicar erros de compilação para a base de dados de bugs. Para erros de compilação use a mailing list.

Para um relatório completo dos bugs e desejos actuais do KDevelop consulte a base de dados. Os relatórios estão separados por componentes, por isso voçe também pode aceder a um Relatório com contagem por Componente e Gravidade.

Para obter uma vista detalhada dos bugs, active cookies no seu browser internet e depois visite a página de configuração de colunas bugzilla e selecione:
  • changeddate
  • severity
  • priority
  • reporter
  • status
  • component
  • version
  • votes
  • Summary (first 60 characters)

E é tudo, apartir de agora todas as suas consultas à base de dados irão ter todos os detalhes que voçe necessita para o KDevelop.
Boa caçada aos bugs! :)
