Esta página concentra toda informação sobre o lançamentos da série KDevelop 1.x

O desenvolvimento do KDE 1.x terminou no dia 11 de Dezembro de 2000 com lançamento do KDevelop 1.3, e nenhum lançamento adicional do KDevelop 1.x está sendo planejado para KDE 1.x.
O KDevelop 1.4 foi desenvolvido para o KDE 2.x e Qt 2.x, mas sugerimos que use o KDevelop 2.1.5 se você precisa de um KDevelop para KDE 2.x.
Atualmente, apenas quatro desejos (wishes) restam no KDevelop 3.0 (para KDE 3.0 e Qt 3.0) dos vários bugs e wishes provindos do KDevelop 1.3.

Compilando a versão 1.3 do repositório CVS

Para conseguir uma versão compilada própria do KDevelop 1.3 a partir das fontes do cvs, faça o checkout do módulo CVS "kdevelop" no :pserver:anonymous@anoncvs.kde.org:/home/kde usando o bugfix CVS branch KDEVELOP_1_0

comando de linha se estiver utilizando o KDE 1.x:
       $ mkdir kdesrc
       $ cd kdesrc
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDEVELOP_1_0 kdevelop
       $ cd kdevelop

No intuito de compilar e instalar o KDevelop (versão CVS) em seu sistema,
digite os comandos abaixo no diretório base da distribuição do KDevelop:

compilando uma versão de lançamento (release): 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (como root)

compilando uma versão de depuração (debug): 
       $ export KDEDIR=/where/your/kde1/is
       $ export QTDIR=/where/qt-1/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (como root)

Bugs conhecidos

Como o desenvolvimento da série de versões 1.x parou, existem muitos bugs conhecidos mas não solucionados. A boa notícia é que todos estes bugs foram corrigidos no KDevelop 3.0. Logo considere utilizar a versão 3.0.
