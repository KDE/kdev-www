Esta página concentra toda informação sobre os lançamentos do KDevelop 2.x

O desenvolvimento parou em 19 de Novembro de 2002 com o lançamento do KDevelop 2.1.4 para KDE 3.0 e nenhum lançamento adicional da série 2.x está planeado. KDevelop 2.1.5 foi uma exceção, lançado em 28 de Janeiro de 2003, como uma atualização para usuários do KDE 3.1. O desenvolvimento se voltou para a nova base de código do KDevelop 3.0 Mais de 80% dos bugs reportados no KDevlop 2.x foram solucionados no KDevelop 3.0.

Compilando a versão 2.1.5 do repositório CVS

Para conseguir uma versão compilada própria do KDevelop 2.1.5 a partir das fontes do cvs, faça o checkout do módulo CVS "kdevelop" no :pserver:anonymous@anoncvs.kde.org:/home/kde usando o bugfix CVS branch KDE_2_2_BRANCH
Você também precisará fazer o checkout do kde-common a partir do cvs.kde.org para obter o diretório admin se você está utilizando o KDE 3.
Preste atenção aos diferentes cvs branches que certas versões KDE 3 requerem:
  • KDE-3.0.x requer o tag KDE_3_0_BRANCH,
  • KDE-3.1.x requer o tag KDE_3_1_BRANCH,
  • apenas o cvs HEAD não precisa de um cvs tag para o kde-common,
este é o motivo pelo qual nesta situação não se usa a opção '-r tagName_seeAbove'.

comando de linha se você está usando o KDE 3:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r tagName_seeAbove kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

comando de linha se você está usando o KDE 2.2:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

No intuito de compilar e instalar o KDevelop (versão CVS) em seu sistema,
digite os comandos abaixo no diretório base da distribuição do KDevelop:

compilando uma versão de lançamento (release): 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (como root)

compilando uma versão de depuração (debug): 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (como root)
Nota: Para o KDE 3 é necessário no mínimo Qt > 3.0.3, para o KDE 2.2.x é necessário Qt 2.3.x
Evite "linkar" o código a ambas versões do Qt, Qt2 e Qt3! Senão a partida do KDevelop irá falhar com um misterioso relatório de travamento...

Plugins and Consertos (patches)

Devido a várias razões (principalmente estabilidade), algumas grandes correções não podem ser aplicadas à versão atual. Mas elas oferecem grandes melhorias, mas você pode baixá-las aqui.
Lembre-se que estas são correções no código fonte, logo você deverá baixar um pacote de fontes compatível, aplicar as correções e por fim recompilá-lo.

Bugs conhecidos
