Ele contém todas as funcionalidades do KDevelop 3.1.x além de:
  • Base de código nova, refeita
    • A base de código do KDevelop foi refeita com o objetivo de melhorar o aproveitamento de código antigo e documentação da biblioteca de base das classes que compõem o KDevelop. Estas classes formam a nova plataforma do KDevelop e apresenta uma API bem documentada
  • Novas ferramentas RAD (Rapid Application Development - Desenvolvimento rápido de Aplicações)
    • A integração KDevDesigner habilita desenhar pela GUI de dentro do KDevelop
    • A integração KDevAssistent provê um rico visualizador de documentação desde o KDevelop
    • O C++ Class Wizard agora pode ser usado para classes do repositório permanente de classes. Isto significa, por examplo, que se a base de dados Code Completion para o Qt for construida então derivar subclasses das classes do Qt é possível desde o Wizard
  • Suporte à linguagem ruby muito aperfeiçoado
    • Novo depurador ruby, combinado com métodos aperfeiçoados de inspeção encadeados no qtruby/korundum, para mostrar metadados do Qt::Object tais como propriedades, enquanto depura
    • O navegador de classe tem um parser aperfeiçoado que mostra sinais e slots Qt e DCOP, métodos attr_accessor e instâncias variáveis
    • Novos modelos de projeto para uma estrutura de aplicação KDE, KConfig XT, servidor DCOP e Qt somente para apps
    • Completa integração com Qt designer/rbuic
  • Outras melhorias e correções
    • O suporte ao controle de versões ( Version Control ) foi refeito. A maior parte das características de integração que estavam desabilitadas no KDevelop 3.1 estão no 3.2, junto com um plugin Subversion muito melhorado
    • Função "Find in files" aperfeiçoada, suporte para Ctags e historia de navigation ajudando a uma navegação mais rápida pelo código
    • Com acréscimo de mais modelosde aplicações
    • Novos modeloss para cmake baseados em apps C/C++ e bibliotecas
    • Novos modeloss para scons-based kde apps
    • veja em página change log para mais detalhes
