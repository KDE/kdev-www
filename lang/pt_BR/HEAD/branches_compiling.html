1. Antes de continuar por favor verifique se seu computador possui o software necessário instalado.

  2. Obtenha o código fonte para o ramo conforme explicado acima.

  3. Inicialize o sistema de compilação:
    $ cd kdevelop
    $ make -f admin/Makefile.common cvs-clean
    $ make -f Makefile.cvs
Após baixar e compilar com sucesso um dos ramos do SVN acima, será preciso mantê-lo atualizado a fim de obter todas as correções.
  1. Para evitar digitar todos os comandos que preparam as variáveis de ambiente toda vez que você for atualizar, você pode colocá-los no arquivo .bashrc ou .cshrc, a dependendo do seu shell.
    • Se você utiliza o interpretador de comandos bash, então as seguintes linhas deverão ser adicionadas:
      export KDEDIR=/onde/seu/kde3/está
      export QTDIR=/onde/seu/qt3/está
      export KDEVELOPDIR=/where/kdevelop/will/be/installed
      export KDEDIRS=$KDEVELOPDIR:$KDEDIR
      export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      export PATH=$KDEVELOPDIR/bin:$QTDIR/bin:$KDEDIR/bin:$PATH
    • Se utiliza tcsh, adicione as seguintes:
      setenv KDEDIR /onde/seu/kde3/está
      setenv QTDIR /onde/seu/qt3/está
      setenv KDEVELOPDIR /where/kdevelop/will/be/installed
      setenv KDEDIRS $KDEVELOPDIR:$KDEDIR
      setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      setenv PATH $KDEVELOPDIR/bin:$QTDIR/bin:$KDEDIR/bin:$PATH
    Note: Por razões obvias você deve editar as três primeiras linhas de acordo com sua configuração! :)


  2. No diretório onde colocou os arquivos do arquivos do kdevelop retirados do SVN execute o comando de atualização:
    $ cd kdevelop
    $ svn up


  3. O próximo passo depende da saída do comando anterior:
    • Se você obteve algo como:
      U /algum_caminho_de_diretorios/Makefile.am
      Você deverá fazer o seguinte:
      $ make -f Makefile.cvs
      seguido do próximo passo ./configure


    • Se obteve algo semelhante a isto:
      P /algum_caminho_de_diretorios/Makefile.am
      Deverá fazer o seguinte:
      $ ./configure --enable-debug=full --prefix=$KDEVELOPDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
      utilizando as opções de ./configure que foram empregadas durante a primeira compilação do KDevelop.


  4. Compile o KDevelop and instale-o:
    $ make
    $ make install (como root, usando su por exemplo)
    Se por alguma razão o comando make falhar, você poderá forçá-lo a ignorar os erros adcionando a opção -k conforme exemplo abaixo:
    $ make -k
    $ make -k install (como root)
    Dependendo da gravidade dos erros obtidos durante a fase de compilação, você terminará com uma versão do KDevelop que poderá ou não rodar. Logo, faça o "make -k install" por sua própria conta e risco!!!. De qualquer forma esses erros desaparecem após algumas horas, e você poderá tentar de novo atualizar e recompilar antes do perigoso comando make install :).


Isto é tudo! Vocé agora possui a versão mais atualizada do KDevelop. Para iniciar basta fazer:
$ kdevelop

Tudo que precisa fazer para mantê-lo autualizado é repetir estes passos toda vez que alguém na kdevelop-devel lista de discussão corriga um bug. Por isso é extremamente recomendado que se junte a esta lista de discussão.

Você deve reportar os bugs que encontrar no formulário de submissão de bugs. Você não deverá relatar erros de compilação na base de dados de bugs. Para erros de compilação use a a lista de lista de discussão.

Você poderá verificar os bugs na base de dados de bugs.
