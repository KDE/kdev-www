Bevor Sie mit der Compilierung Ihrer frisch heruntergeladenen KDevelop Version starten, versichern Sie sich, dass Sie die folgenden Programme installiert haben. Beachten Sie, dass Sie seltsame Fehler erhalten können, wenn Sie ältere Versionen als die angegebenen haben.
Usually you can see which software version is installed in your computer by calling the program with a "--version" argument. For example to find which autoconf you have installed in your machine do: 
   autoconf --version

KDE >= 3.2.0 www.kde.org
Qt >= 3.2.3 www.trolltech.com/products/qt
g++ >= 2.95.3 (oder Kompatible) gcc.gnu.org
GNU make www.gnu.org/software/make
perl >= 5.004 www.perl.com
autoconf >= 2.52 www.gnu.org/software/autoconf
automake >= 1.6 www.gnu.org/software/automake
flex >= 2.5.4 www.gnu.org/software/flex


Diese Werkzeuge sind optional, d.h. KDevelop wird ohne sie funktionieren. Allerdings werden Sie dann einige Features wie das Erzeugen der Dokumentation oder Suche in mehreren Dateien vermissen. Daher sollten Sie sich möglicherweise vergewissern, dass diese Programme installiert sind.

Bitte beachten Sie, dass Sie für die Nutzung der Sprachenunterstützung von KDevelop für Ada, Pascal, Java etc. der entsprechende Kompiler vorhanden sein muss. KDevelop begrenzt nicht die Auswahl der Kompiler, liefert aber für einige von Ihnen Konfigurationsdialoge als Plug-ins mit. Zur Zeit vorhandene Plug-ins sind gcc, g++, f77, pghpf (Portland Group High Performance Fortran Compiler), pgf77 (Portland Group Fortran 77 Compiler), dcc (Borland Delphi Compiler) und fpc (Free Pascal Compiler).
