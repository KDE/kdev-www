Bevor Sie beginnen Ihre soeben heruntergeladene Version von KDevelop zu kompilieren, stellen Sie bitte sicher, dass Sie die folgenden Programme installiert haben. Beachten Sie, dass Sie seltsame Fehlermeldungen erhalten können falls Sie ältere Versionen verwenden.
Normalerweise können sie das Programm mit dem "--version" Argument aufrufen um feststellen, welche Version der Software auf Ihrem Computer installiert ist. Um beispielsweise festzustellen, welche Version von autoconf installiert ist, führen Sie den diesen Befehl aus: 
   autoconf --version

Falls Sie kein autoconf >= 2.52 und automake >= 1.6 haben, können Sie versuchen mit den folgenden Befehlen diese Anforderungen zu umgehen: 
      $  cd KDevelopSourceDirectory
      $  rm -Rf admin
      $  ln -s $KDEDIR/share/apps/kdelibs/admin admin
Falls das nicht funktioniert, kommen Sie um eine Aktualisierung von autoconf und automake nicht herum. Diese Werkzeuge sind optional, d.h. KDevelop funktioniert auch ohne sie. Allerdings werden Sie einigef Funktionen wie das Erzeugen der Dokumentation oder die Suche in mehreren Dateien vermissen. Daher sollten Sie sich möglicherweise vergewissern, dass diese Programme installiert sind.

Bitte beachten Sie, dass Sie für die Sprachünterstützung von ada, pascal, java, etc. den entsprechenden Compiler installiert haben müssen. KDevelop beschränkt Sie nicht in der Compilerauswahl, bietet jedoch Einstellungsdialoge für einige. Derzeit werden Compiler Plugins für gcc, g++, f77, pghpf (Portland Group High Performance Fortran Compiler), pgf77 (Portland Group Fortran 77 Compiler), dcc (Borland Delphi Compiler) und fpc (Free Pascal Compiler) angeboten.
