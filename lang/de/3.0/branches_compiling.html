Laden Sie sich die Quellcodes für den KDE_3_2_BRANCH wie oben beschrieben herunter.
Bevor Sie fortfahren stellen Sie bitte sicher, dass alle benötigte Software installiert ist.

Um KDevelop (CVS-Version) auf Ihrem System zu compilieren und installieren, führen Sie folgende Kommandos aus im Basisverzeichnis aus (wir nehmen an dass Sie eine Bash Shell benutzen).
       $ export KDEDIR=/wo/ihr/kde3/ist
       $ export QTDIR=/wo/ihr/qt3/ist
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ cd kdevelop
       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
Anmerkung: Aus uns unerklährlichen Gründen sollten Sie die ersten zwei Zeilen Ihrer Systemkonfiguration anpassen :-)

Compilieren einer Debug-Version: 
       $ ./configure --enable-debug=full --prefix=$KDEDIR
 --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
       $ make
       $ make install (as root)
Compilieren einer Release-Version: 
       $ ./configure --prefix=$KDEDIR
 --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
       $ make
       $ make install (as root)

Nachdem Sie einen der CVS-Zweige erfolgreich heruntergeladen und compiliert haben, ist es vorteilhaft immer auf dem aktuellsten Stand zu bleiben um alle Patches zu bekommen. Um das andauernde Eintippen der oben genannten Kommandos zu vermeiden, bietet es sich an, die notwendigen Umgebungsvariablen in Ihrer .bashrc oder .cshrc-Datei zu speichern.
Fall Sie eine Bash-Shell benutzen fügen Sie bitte diese Zeilen hinzu:
export KDEDIR=/wo/ihr/kde3/ist
export QTDIR=/wo/ihr/qt3/ist
export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH
Fall Sie tcsh nutzen, fügen Sie bitte diese Zeilen hinzu:
setenv KDEDIR /wo/ihr/kde3/ist
setenv QTDIR /wo/ihr/qt3/ist
setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
setenv PATH $QTDIR/bin:$KDEDIR/bin:$PATH

Anmerkung: Aus uns unerklährlichen Gründen sollten Sie die ersten zwei Zeilen Ihrer Systemkonfiguration anpassen :-)


Als nächstes wechseln Sie in das Verzeichnis in dem Sie die Dateien kdevelops aus dem CVS gespeichert haben und führen dieses Aktualisierungs-Kommando aus:
 $ cd wo_ihr_kde3_quellcode_ist/kdevelop
 $ cvs update -dP

Der nächste Schritt hängt von der Ausgabe des vorherigen Schritts ab:
Falls Sie eine Meldung wie diese bekommen haben:
           U /irgendein_pfad/Makefile.am
... sollten Sie folgendes tun:
 $ make -f Makefile.cvs
und dann alle der folgenden Schritte.

Falls Sie eine Meldung wie diese bekommen haben:
           P /irgendein_pfad/Makefile.am
... sollten Sie folgendes tun:
 $ ./configure --enable-debug=full --prefix=$KDEDIR
 --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
und dann alle der folgenden Schritte.

  $ make
  $ make install (als root)
Falls aus irgendeinem Grund das make Kommando schiefgeht, k&oum;nnen Sie make dazu zwingen Fehler zu ignorieren indem Sie die -k Option hinzufügen. Je nachdem wie schwerwiegend der Fehler ist besteht könnten Sie mit einer funktionierenden oder nicht funktionierenden KDevelop-Installation enden. Die Verwendung der "make -k install" Option ist auf Ihre eigene Verantwortung! Meistens werden diese Fehler nach wenigen Stunden repariert, somit ist es meistens besser ein paar Stunden zu warten anstelle folgende gefährliche Kommandos auszuführen ;-)
 $ make -k
 $ make -k install (als root)
Das war's. Sie sind jetzt im Besitz der aktuellsten Version von KDevelop. Um KDevelop zu starten geben Sie einfach folgendes ein:
 $ kdevelop
Jetzt müssen Sie nur noch diese Schritte jedes Mal wiederholen wenn jemand in der kdevelop-devel Mailingliste einen Bug repariert.

Folgen Sie den normalen Schritten, aber verwenden Sie gmake anstatt von make und nach 
       $ gmake -f Makefile.cvs
geben Sie folgendes ein: 
       $ export CXXFLAGS="-L/usr/X11R6/lib -I/usr/local/include/db41
 -I/usr/local/include -L/usr/local/lib -I/usr/X11R6/include"
Bitte beachten Sie, dass Sie Berkeley DB 4.1 installiert haben müssen, falls Sie eine andere Version verwenden, passen Sie bitte den Pfad entsprechend an.
Ändern Sie außerdem in der Konfigurationsdatei die Zeile "ac_cv_bdb=no" zu "ac_cv_bdb=yes", da ansonsten die DB-Bibliotheken nicht gefunden werden können.
Um herauszufinden welche Version Sie installiert haben: 
       $ pkg_info | grep db41
       db41-4.1.25_1       The Berkeley DB package, revision 4.1
Die FreeBSD Entwickler haben libdb.so in libdbxx.so umbenannt. Deshalb ersetzen Sie bitte "-ldb" mit "-ldb41" in lib/catalog/Makefile.am
Danach können Sie mit dem normalen Vorgehen fortfahren: 
       $ ./configure --prefix=/blah --enable-debug=full
