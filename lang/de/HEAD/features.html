Basierend auf KDevelop 3.2.x wurden folgende Verbesserungen implementiert
  • Zukünftig geplante Verbesserungen finden Sie auf dem KDE 3.5 Feature Plan (Zukunftsplan)
  • Sie können jetzt CMAKE verwenden, um KDevelop zu kompilieren.
  • Die Unterstützung für C++ wurde verbessert
    • Qt4 wird unterstützt, für Qt3.x- und Qt4.x-Projekte wird unterschiedlicher Code generiert
    • Im Optionsdialog von C++ wurde ein neuer Reiter (Tab) hinzugefügt, in dem die Qt-Bibliothek konfiguriert werden kann. Sie können einstellen ob und in welcher Version Qt genutzt werden soll. Auﬂerdem kann der Installationspfad der Qt-Bibliothek eingestellt werden.
  • Der Debugger für C, C++ und Ruby wurde verbessert:
    • Das Haltepunkt-Fenster des Debuggers ist einfacher zu benutzen.
    • Wenn die Anwendung angehalten ist, wird automatisch das Variablen-Fenster angezeigt. Wenn die Anwendung beendet wird wird es automatisch wieder ausgeblendet.
    • Es ist jetzt möglich die Werte von Ausdrücken und Variablen im Variablenfenster zu ändern.
    • Die Variablenansicht wurde verbessert
  • PHP5 wird unterstützt
  • Der Projektassistent wurde verbessert:
    • Er basiert auf der KDevelop-Plattformarchitektur, wo die Projektmanagement-GUI von einem bestimmten Buildsystem getrennt ist. Das Buildsystem ist für den Einsatz von Plugins vorbereitet ("Importassistenten, Builder und Editoren")
    • Arbeitsplatz-Unterstützung ist verfügbar
    • Der Projektassistent kann Builder erstellen
    • Importassistenten für: automake, qmake und eigene makefiles
  • Bitte lesen Sie die Liste der Äderungen (changelog) für mehr Details
