1. Bevor Sie fortfahren, prüfen Sie bitte, ob auf Ihrem Computer die benötigte Software installiert ist.

  2. Holen Sie die Quellen des -Zweiges wie es oben erklärt ist.

  3. Initialisieren Sie das Build-System:
    $ cd kdevelop
    $ make -f admin/Makefile.common cvs-clean
    $ make -f Makefile.cvs
Nachdem Sie einen der SVN-Zweige heruntergeladen und erfolgreich kompiliert haben, wollen Sie sicher auf den neuesten Stand bleiben, um alle Patches zu bekommen.
  1. Um alle Befehle für die nötigen Umgebungsvariablen nicht jedesmal neu eintippen zu müssen, sollten Sie sie in Ihre ~/.bashrc oder ~/.cshrc eintragen.
    • Wenn sie die Bash verwenden, fügen Sie die folgenden Zeilen ein:
      export KDEDIR=/pfad/zu/kde
      export QTDIR=/pfad/zu/qt
      export KDEVELOPDIR=/where/kdevelop/will/be/installed
      export KDEDIRS=$KDEVELOPDIR:$KDEDIR
      export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      export PATH=$KDEVELOPDIR/bin:$QTDIR/bin:$KDEDIR/bin:$PATH
    • Wenn sie tcsh verwenden, fügen Sie die folgenden Zeilen ein:
      setenv KDEDIR /pfad/zu/kde
      setenv QTDIR /pfad/zu/qt
      setenv KDEVELOPDIR /where/kdevelop/will/be/installed
      setenv KDEDIRS $KDEVELOPDIR:$KDEDIR
      setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      setenv PATH $KDEVELOPDIR/bin:$QTDIR/bin:$KDEDIR/bin:$PATH
    Hinweis: Sie sollten die jeweils ersten drei Zeilen Ihrer Konfiguration anpassen! :)


  2. Gehen Sie in das Verzeichnis, wo sie die Dateien für kdevelop von SVN gespeichert haben und führen Sie das Update-Kommando aus:
    $ cd kdevelop
    $ svn up


  3. Der nächste Schritt hängt von der Ausgabe des vorigen Kommandos ab:
    • Wenn Sie so etwas bekommen:
      U /some_directory_path/Makefile.am
      Sollten Sie:
      $ make -f Makefile.cvs
      gefolgt vom nächsten ./configure-Schritt.


    • Wenn Sie so etwas bekommen:
      P /some_directory_path/Makefile.am
      Sollten Sie:
      $ ./configure --enable-debug=full --prefix=$KDEVELOPDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
      mit den ./configure-Optionen, die sie beim ersten Kompilieren benutzt haben, ausführen.


  4. Kompilieren und Installieren von KDevlop:
    $ make
    $ make install (als root)
    Sollte make aus irgendeinem Grund fehlschlagen, können Sie es sämtliche Fehler mit der Option -k ignorieren lassen:
    $ make -k
    $ make -k install (als root)
    Abhängig von der Schwere der Fehler, haben Sie am Ende eine funktionierende oder eine nicht funktionierende Version von KDevelop. Benutzen Sie das "make -k install" Kommando auf Ihre Verantwortung!!! In jedem Fall sollten diese Fehler nach einigen Stunden weggehen, so dass Sie vielleicht ein neues Update probieren sollten, bevor Sie zu "make -k" greifen.


Das war's! Sie haben jetzt die neueste Version von KDevelop. Um sie zu starten tippen Sie:
$ kdevelop

Alles, was Sie tun müssen, damit sie auf dem neusten Stand bleiben, ist, diese Schritte zu wiederholen, wenn jemand auf der kdevelop-devel-Mailingliste einen behobenen Bug meldet. Es ist also sehr empfehlenswert, dieser Liste beizutreten.

Bitte melden Sie alle Bugs, die Sie finden, mit dem Bug-Meldeformular. Melden Sie jedoch keine Kompilationsfehler an die Bugdatenbank. Benutzen Sie dafür bitte die Mailingliste.

Sie können sich die Bugs in der Bug-Datenbank ansehen.
