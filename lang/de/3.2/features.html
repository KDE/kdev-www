Beinhaltet alle KDevelop 3.1.x Features plus:
  • Neue überarbeitete Code-Basis
    • Die KDevelop Code-Basis wurde überarbeitet mit dem Ziel die Code-Wiederverwendbarkeit zu verbessern und die Basis-Bibliotheksklassen aus denen KDevelop besteht zu dokumentieren. Diese Basisklassen bilden die neue KDevelop Plattform und stellen eine übersichtliche und gut dokumentierte API dar
  • Neue RAD (Rapid Application Development - Schnelle Anwendungsentwicklung) Werkzeuge
    • KDevDesigner Integration macht GUI Design direkt aus KDevelop möglich
    • KDevAssistent Integration liefert einen vielfältigen Dokumentationsbetrachter direkt in KDevelop
    • Der C++ Klassen Wizard kann nun für Klassen im ständigen Klassenspeicher benutzt werden. Das bedeutet z.B., dass, wenn die Code-Completion Datenbank für Qt erstellt wird, ein Subclassing von Qt-Klassen im Wizard möglich ist
  • Deutlich verbesserte Ruby Sprachunterstützung
    • Neuer Ruby Debugger, kombiniert mit verbesserten Untersuchungsmethoden in den qtruby/korundum Bindings, um Qt::Object Metadaten, wie Eigenschaften, während des Debuggens anzuzeigen
    • Der Kassenbrowser hat einen verbesseren Parser, der Qt und DCOP Signale und Slots, attr_accessor Methoden und Instanzvariablen anzeigt
    • Neue Projekt-Templates für ein KDE Anwendungsframework, KConfig XT, DCOP Server und reine Qt Anwendungen
    • Volle Qt Designer/rbuic Integration
  • Andere Verbesserungen und Bugfixes
    • Die Versionskontrollunterstützung wurde überarbeitet. Die Meisten der Integrationsfeatures die in KDevelop 3.1 abgeschaltet waren sind in 3.2 zurück, zusammen mit einem deutlich verbesserten Subversion plugin.
    • Verbessertes "Suchen in Dateien", Ctags unterstützung und History-Navigation ermöglichen schnelleres navigieren im Code
    • Es stehen mehr Anwendungstemplates zur verfügung
    • Neue Templates für cmake basierte C/C++ Anwendungen und Bibliotheken
    • Neue Templates für scons basierte KDE Anwendungen
    • Mehr Details finden sie auf der Change-Log Seite
