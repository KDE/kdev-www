Diese Seite fasst alle Informationen zur KDevelop 2.x

Die Entwicklung wurde am 19. November 2002 mit der Veröffentlichung von KDevelop 2.1.4 für KDE 3.0 eingestellt. Es sind keine weiteren 2.x Versionen geplant. KDevelop 2.1.5 war eine Ausnahme, veröffentlicht am 28. Januar 2003, als eine Upgradeversion für KDE 3.1 Nutzer. Die Entwicklung wurde in die neuere KDevelop 3.0 Codebasis verlagert. Mehr als 80 Prozent der in KDevelop 2.x gemeldeten Bugs sind in KDevelop 3.0 behoben.

Version 2.1.5 vom CVS-Verzeichnis kompilieren

Um ein selbstkompiliertes KDevelop 2.1.5 aus den CVS Quelltexten zu erhalten, besorgen Sie sich den Bugfix CVS Zweig KDE_2_2_BRANCH des Moduls "kdevelop" von :pserver:anonymous@anoncvs.kde.org:/home/kde
Sie benötigen auch kde-common von cvs.kde.org um das admin-Verzeichnis zu erhalten, falls Sie KDE 3 verwenden.
Bitte beachten Sie die verschiedenen CVS Zweige die die Verschiedenen KDE 3-Versionen benötigen:
  • KDE-3.0.x benötigt den Tag KDE_3_0_BRANCH,
  • KDE-3.1.x benötigt den Tag KDE_3_1_BRANCH,
  • nur CVS HEAD benötigt keinen Tag für kde-common,
deshalb benutzen Sie in diesem Fall die Option '-r tagName_sieheOben' nicht.

Kommandozeile falls Sie KDE 3 benutzen:c
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r tagName_sieheOben kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

Kommandozeile falls sie KDE 2.2 einsetzen:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Um KDevelop (CVS-Version) auf Ihrem System zu installieren, führen Sie bitte die folgenden Befehle vom Wurzelverzeichnis Ihrer KDevelop Distribution aus:

eine für die Veröffentlichung kompilierte Version: 
       $ export KDEDIR=/wo/Ihr/KDE2/oder/KDE2/Verzeichnis/ist
       $ export QTDIR=/wo/Ihr/qt-2.3.1/oder/qt-3.0.x/ist
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

eine mit Debuginformationen kompilierte Version: 
       $ export KDEDIR=/wo/Ihr/KDE2/oder/KDE2/Verzeichnis/ist
       $ export QTDIR=/wo/Ihr/qt-2.3.1/oder/qt-3.0.x/ist
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (als root)
Hinweis: Für KDE 3 benötigen Sie mindestens Qt > 3.0.3 (besser KDE's qt-Kopie), für KDE 2.2.x benötigen Sie Qt 2.3.x
Vermeiden Sie beide Qt Versionen, Qt2 und Qt3 zu linken. Anderenfalls wird ein Start von KDevelop mit einem seltsamen Absturzreport fehlschlagen.

Plugins und Patches

Wegen verschiedener Gründe (hauptsächlich Stabilität), können einige größere Patches nicht zur aktuellen Version hinzugefügt werden. Da diese dennoch hervorragende Erweiterungen bieten, können Sie diese hier herunterladen. Bitte beachten Sie dass es sich um Quelltextpatches handelt, d.h. Sie müssen ein passendes Quelltextpaket herunterladen, es patchen und neu kompilieren.
Viel Spaß!

Ereignisbehandlung für Message, Breakpoint und Watch Fenster (wichtig für KDE/QT-3.0.x) Patch		 Iztok (I (in) siol .. net) 2.1 und die entsprechenden Bugfix Versionen (2.1.x), aber nur für KDE-3 Das originale KDevelop 2.1.5 (KDE_2_2_BRANCH) ist nicht vollständig in den Erzeugungsoptionen für C++-Konsolenprojekte. Die Builds wurden nicht in den entsprechenden Unterverzeichnissen vorgenommen, die Buildkonfigurationen konnten nicht entfernt werden und das Debugging von diesen funktionierte gänzlich nicht. Dieser Patch behebt dies. Ich betreue und entwickle viele Projekte für ix86- und arm-Prozessoren. Die Arbeit mit diesem Patch funktioniert reibungslos für bereits mindestens ein halbes Jahr. Die Konfiguration beginnt mit Optionen-> KDevelop Setup->Compiler wo die entsprechenden Compilerpfade angepasst werden sollten. Ich persönlich ziehe die Handhabung der Buildkonfigurationen in KDevelop2 der in KDevelop3 vor, da es die Compilerinformationen von den Projekteinstellungen getrennt, umschaltbar auf verschiedenen Hosts.
Verbesserungen in der Cross-Plattform Entwicklungsunterstützung +
Ereignisbehandlung für Message, Breakpoint und Watch Fenster (wichtig für KDE/QT-3.0.x) +
Trennung der Datenquellen von KDevelop 3		 Iztok (I (in) siol .. net) 2.1 und die entsprechenden Bugfix Versionen (2.1.x), aber nur für KDE-3 Da Gideon (KDevelop3) in KDevelop umgenannt wurde, traten die Datenquellen in Konflikt mit denen von KDevelop 2. Dieser Patch löst bewirkt folgendes::
  • Das kdevelop Binary wird in kdevelop2 umbenannt
  • Die kdevelop2.desktop und kdevelop2-setup.desktop Dateien werden zum kdevelop/ Verzeichnis hinzugefügt
  • Das $KDEDIR/share/..../kdevelop Verzeichnis wird in kdevelop2 umbenannt
  • Das $HOME/.kde/share/..../kdevelop Verzeichnis wird in kdevelop2 umbenannt
  • Der Quelltext ist auf kdevelop2 hin abgeändert, sodass es für die Ressourcen in den richtigen Ordnern sucht.
  • KDevelop2 kann mit Hilfe von KDE_3_1_BRANCH und KDE_3_2_BRANCH erzeugt werden, eindeutig wann das entsprechende admin Verzeichnis verwendet wird
