Questa pagina raccoglie tutte le informazioni sulla serie di rilasci di KDevelop 2.x

Lo sviluppo è stato fermato il 19 novembre 2002 con il rilascio di KDevelop 2.1.4 per KDE 3.0, non sono previsti altri rilasci per la serie 2.x. KDevelop 2.1.5 è stata un'eccezione, rilasciato il 28 gennaio 2003 come versione di aggiornamento per gli utenti di KDE 3.1. Lo sviluppo è stato spostato nella nuova base di codice di KDevelop 3.0. Più dell'80 percento dei bugs riportati per KDevelop 2.x sono stati risolti in KDevelop 3.0.

Le istruzioni si possono trovare su Come Installare KDevelop 2.1.5 su Suse 8.2 (28KB .txt). Questo spiega anche come installare i riferimenti C/C++.

Compilare la versione 2.1.5 dall'archivio CVS

Per compilare una versione di KDevelop 2.1.5 dai sorgenti CVS fai il checkout del modulo CVS "kdevelop" su :pserver:anonymous@anoncvs.kde.org:/home/kde usando il ramo (branch) CVS di bugfix KDE_2_2_BRANCH
Dovrai fare checkout anche di kde-common da cvs.kde.org per avere la directory admin se stai usando KDE 3.
Fai attenzione ai differenti rami (branch) CVS che alcune versioni di KDE 3 richiedono:
  • KDE-3.0.x richiede il tag KDE_3_0_BRANCH,
  • KDE-3.1.x richiede il tag KDE_3_1_BRANCH,
  • HEAD non ha bisogno di un tag cvs per kde-common,
ecco perchè in questo cosa non usi l'ozione '-r nomeTag_vediSopra'.

linea di comando se usi KDE 3:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r nomeTag_vediSopra kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

linea di comando se usi KDE 2.2:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Per compilare e installare KDevelop (versione CVS) sul tuo sistema,
scrivi i comandi qui sotto nella directory di base della distribuzione di KDevelop:

per una versione di release: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

per una versione di debug: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)
Nota: Per KDE 3 hai bisogno almeno di Qt > 3.0.3 (meglio la copia Qt di KDE), per KDE 2.2.x hai bisogno di Qt 2.3.x
Evita di linkare a entrambe le versioni di Qt, Qt2 e Qt3! Altrimenti l'avvio di KDevelop fallirà con un misterioso crash report...
