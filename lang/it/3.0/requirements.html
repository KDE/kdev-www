Prima di cominciare a compilare la tua versione di KDevelop scaricata di recente, assicurati che i seguenti programmi siano installati. Ricorda che puoi ricevere strani errori, se usi vecchie versioni.
Generalmente puoi vedere la versione del software installato nel tuo computer richiamando il programma con l'argomento "--version". Per esempio per trovare quale autoconf hai installato nella tua macchina, fai: 
   autoconf --version

Potete aggirare le esigenze di autoconf >= 2.52 e automake >= 1.6, facendo: 
      $  cd KDevelopSourceDirectory
      $  rm -Rf admin
      $  ln -s $KDEDIR/share/apps/kdelibs/admin admin
Se questo non funziona, allora avete bisogno di aggiornare autoconf e automake. If that doesn't work, then you really need to update autoconf and automake. Questi tools sono opzionali, questo significa che KDevelop potra` lavorare senza essi. Ma gli mancheranno delle funzionalita` come la generazione di documenti o la ricerca in alcuni files. Cosi` dovreste probabilmente verificare il vostro sistema per questi programmi.

Si noti che per usare il supporto ai linguaggi in KDevelop per ada, pascal, java, etc. necessiti del corrispondente compilatore. KDevelop non limita il compilatore che dovreste usare, ma offre un dialog per le opzioni di alcuni di loro. Al momento sono disponibili i plugin per gcc, g++, f77, pghpf (Portland Group High Performance Fortran Compiler), pgf77 (Portland Group Fortran 77 Compiler), dcc (Borland Delphi Compiler) and fpc (Free Pascal Compiler).
