Include tutte le caratteristiche di KDevelop 3.1.x oltre a:
  • Codice di base riformattato
    • L'insieme del codice di KDevelop é stato riformattato con lo scopo di migliorare il riutilizzo del codice e la documentazione della libreria di base. Queste nuove classi sono alla base della nuova piattaforma di KDevelop e sono accompagnate da una API sintetica e ben documentata.
  • Nuovi strumenti per il RAD (Rapid Application Development)
    • l'integrazione di KDevDesigner permette il design della GUI design all'interno di KDevelop stesso
    • l'integrazione di KDevAssistent fornisce la possibilità di dare un'occhiata alla documentazione all'interno di KDevelop
    • Il Wizard per le classi C++ può ora essere usato per le classi facenti parte all'insieme persistente. Questo significa , ad esempio, che se il database per il Completamento del Codice per le Qt é stato costruito, allora é resa possibile nel Wizard la creazione delle sottoclassi a partire da classi Qt
  • Supporto per Ruby notevolmente migliorato
    • il nuovo debugger per Ruby e la vista migliorata dei metodi all'interno dei qtruby/korundum bindings, ora permettono di mostrare i metadata Qt::Object come proprietá durante il debug
    • Il browser delle classi ha un parser migliorato che mostra i segnali Qt e DCOP oltre agli slots, ai metodi attr_accessor e alla variabili d'istanza
    • Nuovi template di progetto per lo schema di un'applicazione KDE, KConfig XT, DCOP server e applicazioni basate esclusivamente su Qt
    • Completa integrazione del designer Qt/rbuic
  • Altri miglioramenti e bugfixes
    • Il supporto per il Controllo di Versione é stato riformattato. Gran parte delle features di integrazione che erano state disabilitate in KDevelop 3.1 sono state riprese nella versione 3.2, insieme ad un plugin molto migliorato per Subversion.
    • Migliorati "Trova nei files"e il supporto e navigazione della storia di Ctags, che ora forniscono una navigazione piú veloce del codice
    • Sono forniti anche molte altri template per le applicazioni
    • Sono disponibili nuovi template per applicazioni C/C++ basate su cmake e per le librerie
    • Sono disponibili nuovi template per applicazioni kde basate su scons
    • leggi la pagina dei changelog per ulteriori dettagli
