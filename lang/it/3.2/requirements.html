Prima di tentare la compilazione del vostro KDevelop appena scaricato, assicuratevi di aver istallato nel vostro sistema i seguenti programmi. Tenete presente che potreste avere strani errori usando versioni più vecchie.
Normalmente puoi vedere che versione di software è installata sul tuo computer chiamando il programma con l'argomento "--version". Per esempio per sapere quale versione di autoconf hai installata fai:: 
   autoconf --version

Programma/Libreria Sorgente
KDE >= www.kde.org
Qt >= 3.2.3 www.trolltech.com/products/qt
g++ >= 2.95.3 (or compatible) gcc.gnu.org
GNU make www.gnu.org/software/make
perl >= 5.004 www.perl.com
autoconf >= 2.52 www.gnu.org/software/autoconf
automake >= 1.6 www.gnu.org/software/automake
flex >= 2.5.4 www.gnu.org/software/flex
libtool >= 1.4 www.gnu.org/software/libtool/libtool.html


Questi altri strumenti sono opzionali, il che significa che KDevelop funziona anche senza. Ma noterete la mancanza di alcune caratteristiche quali la generazione di documentazione o la ricerca su più file. Quindi è probabile che preferiate installare questi pacchetti.

Nota che per usare il supporto di KDevelop per i linguaggi ada, pascal, java, etc. avrai bisogno del corrispondente compilatore del linguaggio, KDevelop non pone limiti a quale copilatore usi ma offre dialogs per le opzioni di compilazione per alcuni di questi. Attualmente i plugins per compilatori disponibili sono gcc, g++, f77, pghpf Portland Group High Performance Fortran Compiler), pgf77 (Portland Group Fortran 77 Compiler), dcc (Borland Delphi Compiler) and fpc (Free Pascal Compiler).
