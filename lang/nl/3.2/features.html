Het bevat alle KDevelop 3.1.x functies plus:
  • Nieuwe herschreven codebase
    • De KDevelop codebase is herschreven met het doel de code hergebruiking te verbeteren en de base library classes, die KDevelop maken, te documenteren. Deze basis klassen vormen het nieuwe KDevelop platform en presenteren een beknopt en goed gedocumenteerd API
  • Nieuwe RAD (Rapid Application Development) tools
    • KDevDesigner integratie laat GUI design vanuit KDevelop zelf toe
    • KDevAssistent integratie zorgt voor een eigenschap rijke documentatie viewer vanuit KDevelop
    • The C++ Class Wizard kan nu gebruikt worden voor classes in de persistant class store. Dit betekent bijvoorbeeld dat wanneer de code completion database voor Qt is gemaakt de subclassing van Qt classes mogelijk is in de wizard.
  • Goed verbeterde ruby language support
    • Nieuwe ruby debugger, gecombineerd met verbeterde inspecteer methoden in de qtruby/korundum bindings, Om Qt::Object metadata zoals eigenschappen te tonen, tijdens het debuggen
    • Class browser heeft een verbeterde parser die QT en DCOP signalen, slots, attr_accessor methoden en instantie variabelen toont.
    • Nieuwe project templates voor een KDE applicatie framework, KConfig XT, DCOP server en Qt only apps
    • Full Qt designer/rbuic integratie
  • Andere verbeteringen en bugoplossingen
    • De versie controle support is opnieuw gemaakt. De meeste integratie eigenschappen die uitgeschakeld waren in KDevelop 3.1 zijn terug in 3.2, samen met een erg verbeterde Subversie plugin.
    • Verbeterde "Find in files", Ctags support en history navigatie zorgen voor snellere code navigatie
    • Meer applicatie templates zijn bezorgd
    • nieuwe templates voor cmake gebasseerde C/C++ apps en bibliotheken
    • nieuwe templates voor scons-gebasseerde kde apps
    • zie de change log pagina voor meer details
