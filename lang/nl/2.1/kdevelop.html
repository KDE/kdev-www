Deze pagina brengt alle informatie over de KDevelop 2.x release serie bijeen

Ontwikkeling is stopgezet op 19 november 2002 met de KDevelop 2.1.4 release voor KDE 3.0, en geen verdere 2.x releases staan geplanned. KDevelop 2.1.5 was een uitzondering, uitgebracht 28 januari 2003, als een upgrade versie voor KDE 3.1 gebruikers. Ontwikkeling gaat verder met de nieuwe KDevelop 3.0 code base. Meer dan 80% van de gerapporteerde bugs in KDevelop 2.x zijn opgelost in KDevelop 3.0.

Hoe KDevelop 2.1.5 Te Installeren

Instructies betreffende het installeren van KDevelop 2.1.5 op Suse 8.2 (28KB .txt) zijn beschikbaar. Het legt ook uit hoe je de C/C++ naslagwerken installeert.

Versie 2.1.5 uit de CVS repository compileren

Om een zelf-gecompileerde KDevelop 2.1.5 uit de CVS broncode te verkrijgen, haal de CVS module "kdevelop" op van :pserver:anonymous@anoncvs.kde.org:/home/kde gebruik makend van de bugfix CVS branch KDE_2_2_BRANCH
Je moet ook kde-common ophalen van cvs.kde.org voor de admin directory als je KDE 3 gebruikt.
Let op welke versie van KDE 3 de verschillende CVS branches nodig hebben:
  • KDE-3.0.x heeft de tag KDE_3_0_BRANCH nodig,
  • KDE-3.1.x heeft de tag KDE_3_1_BRANCH nodig,
  • alleen cvs HEAD heeft geen cvs tag nodig voor kde-common,
en dus gebruik je niet de optie '-r tagName_zieBoven' in de laatste situatie.

Commando's als je KDE 3 gebruikt:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r tagName_zieBoven kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

Commando's als je KDE 2.2 gebruikt:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Om de CVS versie van KDevelop te compileren en te installeren op je systeem
typ je de volgende commando's in in de hoofddirectory van de KDevelop distributie:

een release-gecompileerde versie: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

een debug-gecompileerde versie: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (als root)
Let op: Voor KDE 3 heb je op zijn minst Qt > 3.0.3 (of beter: KDE's qt-copy) nodig, voor KDE 2.2.x heb je Qt 2.3.x nodig.
Link niet met zowel Qt2 als Qt3! Anders zal KDevelop op mysterieuze wijze crashen bij de start...

Plugins en patches

Er zijn verschillende redenen (voornamelijk stabiliteit) dat een aantal van de grotere patches niet kunnen worden aangebracht op de daadwerkelijke versie. Maar ze leveren wel veel verbeteringen op, dus kun je ze hier toch downloaden.
Onthou dat dit broncode patches zijn, dus je moet een compatible broncode package downloaden, dit patchen en dan opnieuw compileren.

De builds worden niet uitgevoerd in de juiste subdirectories, de build configuraties kunnen niet verwijderd worden en het debuggen ervan lukt niet goed. Deze patch lost dit probleem op. Ik onderhoud en ontwikkel nog steeds een groot aantal projecten gericht op ix86 en arm processoren. Dit werk gaat prima met deze patch gedurende al minstens een half jaar. Ik moet zeggen dat ik de afhandeling van build configuraties in KDevelop 2 prefereer boven die in KDevelop 3 omdat het de compiler informatie gescheiden houdt van de project instellingen, zodat het uitwisselbaar is.

event handling van de Message, Breakpoint en Watch windows (belangrijk voor KDE/QT-3.0.x) +
scheiding van de data resources van de KDevelop 3 patch		 Iztok (I (in) siol .. net) 2.1 en zijn bugfix versies (2.1.x), maar alleen voor KDE 3 Aangezien Gideon (KDevelop 3) hernoemd is naar KDevelop, raken zijn data resources in de knoop met die van KDevelop 2. Deze patch lost de volgende problemen op:
  • kdevelop binary is hernoemd naar kdevelop2
  • kdevelop2.desktop en kdevelop2-setup.desktop bestanden zijn toegevoegd aan de kdevelop/ directory
  • $KDEDIR/share/..../kdevelop directories zijn hernoemd naar kdevelop2
  • $HOME/.kde/share/..../kdevelop directory is hernoemd naar kdevelop2
  • Codebase is veranderd voor kdevelop2 zodat het zijn data resources in de juiste directories zoekt
