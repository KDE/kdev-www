1. Sebelum dilanjutkan, tolong periksa dulu kalau software-software yang dibutuhkan sudah di install di komputer anda.

  2. Dapatkan kode asal KDE_3_3_BRANCH branch seperti yang sudah dijelaskan diatas.

  3. Memulai the build system:
    $ cd kdevelop
    $ make -f admin/Makefile.common cvs-clean
    $ make -f Makefile.cvs
Setelah anda men-download dan meng-kompile salah satu dari cabang CVS diatas dengan sukses, anda harus rajin meng-update supaya bisa mendapatkan semua patch.
  1. Untuk menghindari kerepotan yang ditimbulkan kalau anda mengetikkan semua pernyataan yang dibutuhkan untuk mengatur kondisi veriable setiap kali anda meng-update, anda bisa mengatur semua pernyataan itu di .bashrc atau .cshrc file.
    • Kalau anda menggunakan bash shell, tambahkan beberapa baris berikut ini:
      export KDEDIR=/where/your/kde3/is
      export QTDIR=/where/your/qt3/is
      export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH
    • Kalau anda menggunakan tcsh, tambahkan beberapa baris berikut ini:
      setenv KDEDIR /where/your/kde3/is
      setenv QTDIR /where/your/qt3/is
      setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      setenv PATH $QTDIR/bin:$KDEDIR/bin:$PATH
    Catatan: Untuk alasan yang cukup mudah dilihat, anda harus meng-edit baris pertama dan kedua supaya sesuai dengan configuration anda! :)


  2. Selanjutnya, pergi ke directory dimana anda simpan kdevelop dari CVS files dan perintahkan untuk meng-update:
    $ cd kdevelop
    $ cvs update -dP


  3. Langkah selanjutnya tergantung dari hasil perintah sebelumnya:
    • Kalau anda mendapatkan sesuatu seperti:
      U /some_directory_path/Makefile.am
      lakukan yang berikut ini:
      $ make -f Makefile.cvs
      lanjutkan dengan langkah ./configure


    • Kalau anda mendapatkan sesuatu seperti:
      P /some_directory_path/Makefile.am
      lakukan yang berikut ini:
      $ ./configure --enable-debug=full --prefix=$KDEDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
      gunakan ./configure options seperti yang anda gunakan pertama kali anda meng-kompilasi.


  4. Kompile kdevelop dan install:
    $ make
    $ make install (as root)
    Kalau karena alasan tertentu perintah "make" gagal, anda bisa memaksakan "make" untuk mengabaikan errors dengan menambahkan -k option, seperti ini.
    $ make -k
    $ make -k install (as root)
    Tergantung dari beratnya error tersebut, anda mungkin pada akhirnya mendapatkan versi kdevelop yang bekerja atau tidak bekerja. Maka dari itu sadarilah resiko dengan melakukan "make -k install" !!! Biasanya errors tersebut pergi begitu saja setelah beberapa jam, jadi anda mungkin bisa mencoba meng-update lagi sebelum mencoba "make" command :).


Selesai! Anda sekarang mempunyai versi KDevelop yang paling up to date. Untuk memulai KDevelop, ketikkan:
$ kdevelop

Yang harus anda lakukan untuk tetap up to date adalah mengulangi langkah-langkah tersebut setiap kali seseorang di kdevelop-devel milis memperbaiki bug. Maka dari itu, anda disarankan untuk bergabung di milis.

Anda bisa melaporkan bugs yang anda temukan dengan formulir pelaporan bug (bug submition form). Anda tidak seharusnya melaporkan error kompilasi ke bug database. Untuk melaporkan error kompilasi, gunakan milis.

Anda bisa melikat bug-bugs yang ada di bug database.
