Halaman ini memfokuskan ke semua informasi tentang KDevelop 2.x rilis

Pengembangan sudah diberhentikan pada tanggal 19 November 2002 dengan KDevelop 2.1.4 rilis untuk KDE 3.0, dan tidak ada rencana untuk melanjutkan 2.x rilis. KDevelop 2.1.5 adalah pengecualian, dirilis pada tanggal 28 Januari 2003 sebagai versi upgrade untuk pemakai KDE 3.1. Pengembangan sudah berpindah ke code basis yang lebih baru KDevelop 3.0. Lebih dari 80 percent bugs yang dilaporkan di KDevelop 2.x sudah diperbaiki di KDevelop 3.0.

Meng-kompile versi 2.1.5 dari gudang CVS (CVS repository)

Untuk mendapatkan kompile-sendiri (self-compiled) KDevelop 2.1.5 dari source cvs, check module CVS "kdevelop" di :pserver:anonymous@anoncvs.kde.org:/home/kde dengan menggunakan the bugfix CVS branch KDE_2_2_BRANCH
Anda juga perlu memeriksa kde-common dari cvs.kde.org untuk mendapatkan admin directory kalau anda menjalankan KDE 3.
Perhatikan cabang-cabang cvs yang berbeda. Versi KDE 3 tertentu membutuhkan:
  • KDE-3.0.x membutuhkan the tag KDE_3_0_BRANCH,
  • KDE-3.1.x membutuhkan the tag KDE_3_1_BRANCH,
  • hanya cvs HEAD yang tidak membutuhkan cvs tag untuk kde-common,
itulah sebabnya dalam kasus itu jangan menggunakan option '-r tagName_seeAbove'.

command line kalau anda menggunakan KDE 3:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r tagName_seeAbove kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

command line kalau anda menggunakan KDE 2.2:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Dengan tujuan untuk meng-kompile dan meng-install KDevelop (versi CVS) di sistem anda,
ketikan perintah-perintah (commands) dibawah ini di base directory KDevelop distribution:

a release-compiled version: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

a debug-compiled version: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)
Catatan: Untuk KDE 3 anda memerlukan paling tidak Qt > 3.0.3 (lebih baik KDE's qt-copy), untuk KDE 2.2.x anda memerlukan Qt 2.3.x
Hindari penghubungan (linking) kedua versi Qt, Qt2 dan Qt3! Kalau tidak, KDevelop akan gagal sewaktu dimulai dengan laporan crash yang misterius...

Plugins and patches

Karena beberapa alasan yang berbeda (terutama stabilitas program), beberapa patches besar tidak bisa digunakan di versi aktual. Tapi mereka menawarkan perbaikan besar, anda bisa men-downloadnya disini.
Ingat kalau mereka adalah source patches, jadi anda harus men-download paket source yang sesuai (a compatible source package), patch dan kompile kembali.

Classparser patch August Hoerandl + bugfix by joerg.pauly at psystems.org 2.1 and its bugfix versions (2.1.x) Patch ini membolehkan "the classparser" menguraikan (parse) the base libraries (Qt, KDE). Builds belum dijalankan di subdirectories yang cocok, the build configuration belum bisa di hapus dan debugging disitu tidak berjalan sama sekali. Patch ini memperbaiki masalah yang ada. Saya memelihara dan masih mengembangkan banyak sekali proyek-proyek yang menargetkan ix86 dan arm processors. Pekerjaan yang berkaitan dengan patch ini sudah berjalan dengan lancar untuk paling sedikit setengah tahun. Konfigurasi mulai dengan the Options->KDevelop Setup->Compiler dimana compiler path yang cocok seharusnya di-configured. Saya harus mengakui kalau saya lebih suka handling di KDevelop2 build configurations daripada handling di KDevelop3 karena KDevelop2 build configurations memisahkan informasi compiler dari settings proyek, switchable accross the host switch.
