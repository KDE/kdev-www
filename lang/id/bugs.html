Untuk melaporkan bugs (cacat dalam program) atau mengajukan permintaan fungsi-fungsi baru di KDevelop, kunjungi the bug report wizard yang berada di database bugzilla. Anda tidak seharusnya melaporkan error compilasi ke bug database. Untuk melaporkan error compilasi, gunakan milis kami.

Untuk melihat laporan yang lengkap mengenai bugs dan permintaan fungsi-fungsi baru, lihat the bug database. Laporan-laporan tersebut sudah dipisahkan ke dalam bagian-bagian tertentu, sehingga anda juga bisa melihat Banyaknya laporan berdasarkan bagian tertentu dan parahnya bugs (sevirity).

Untuk mendapatkan penampilan menarik sewaktu menampilkan perincian bugs, aktifkan cookies di web browser anda lalu kunjungi halaman konfigurasi bugzilla column dan pilih:
  • tanggal penggantian (changeddate)
  • tingkat keparahan bugs (severity)
  • prioritas (priority)
  • pelapor (reporter)
  • status
  • bagian (component)
  • versi (version)
  • suara pilih (votes)
  • rangkuman (Summary) (60 karakter pertama)

Segitu saja langkah-langkah untuk melaporkan bugs, mulai dari sekarang semua laporan bugs akan mempunyai rincian-rincian yang anda perlukan untuk KDevelop.
Selamat memburu bugs! :)
