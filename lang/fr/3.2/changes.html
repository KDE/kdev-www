  • NOUVEAU - Caractéristiques de KDevelop dans le plan des caractéristiques pour KDE 3.4
  • NOUVEAU - La plate-forme KDevelop 
    • La première sortie de la « plate-forme KDevelop » - une architecture pour contruire des applications IDE qui utilise des modules externes communs et un shell
    • KDevelop IDE et KDevelop Assistant sont maintenant deux applications construites sur la plate-forme KDevelop
    • Architecture retravaillée - les interfaces ont été revues et des profils de modules externes sont introduits
  • AMÉLIORÉ - Le shell KDevelop :
    • Le shell est modifié pour utiliser les profils de modules externes
    • Des notification de changements externes sur les fichiers sont ajoutées
    • La fenêtre principale n'est affichée qu'après l'initialisation
  • AMÉLIORÉ - La documentation API :
    • La documentation de l'API du programme KDevelop est maintenant à http://www.kdevelop.org/HEAD/doc/api/html/index.html
  • AMÉLIORÉ - Support du language Ruby :
    • Un débogueur Ruby est intégré
    • AMÉLIORÉ l'analyseur ruby - plus d'information disponible dans la vue des classes
    • Ajout de modèles d'applications Qt et KDE
  • CORRIGÉ - Petites bogues dans le support du language Perl
  • AMÉLIORÉ - support pour le language Cpp :
    • La fenêtre des points d'arrêt du débogueur est plus facile à utiliser
    • Les caractéristiques Permet/Supprime fonctionnent maintenant correctement
    • Correction du problème qui supprimait les points d'arrêt lorsque le débogueur était lancé pour la première fois
    • Le menu contextuel est plus propre
    • Switch header/implementation can now optionally try to match the current function declaration/definition
    • La base de données de complétion de code peut être supprimée du dialogue de configuration
  • AMÉLIORÉ - Support du language PHP : améliorations dans la complétion du code
  • AMÉLIORÉ - Partie scripts :
    • NOUVEAU un module externe de scripts est ajouté
    • KScript based scripting is supported using KScriptInterface
    • KJSEmbed, Shell et Kommander peuvent être utilisés pour lancer un script
  • AMÉLIORÉ - Output view:
    • Speed optimizations (approx. 8%)
    • Jump to the error location now works for all cases
  • AMÉLIORÉ -Find In Files: Old search results can be stored besides the current search
  • AMÉLIORÉ - Code snippets plugin: Snippets can be divided into groups
  • AMÉLIORÉ - Application templates:
    • Installation, instantiation and compilation is CORRIGÉed for following application templates: wxhello, kxt, kscreensaver, kdevpart, kdevpart2, kofficepart, kateplugin
    • KDevelop plugin and language support templates were updated to match changes in the architecture
    • All occurences of wrong variable substitution in templates are CORRIGÉed
    • CMake application templates added
  • AMÉLIORÉ - Visualisation des différences :
    • Kate part is excluded from a list of difference viewers
    • KDiff3 est maintenant supporté
  • AMÉLIORÉ - Source formatter: Preview for custom style is implemented
  • AMÉLIORÉ - gestionnaire de projet générique :
    • Un autre gestionnaire de projet générique est commencé. Il est basé sur la nouvelle architecture où l'interface de gestinnaire de projet est abstraite pour un syst`me de construction. Les syst`mes de construction gérés en utilisant les modules externes ("project importers, builders et editors")
    • Le support workspace support est disponible
    • Construction de projets : construction avec make
    • Import de projets : automake, makefiles personnalisées
  • AMÉLIORÉ - gestionnaire de projet QMake :
    • L'interface est simplifiée
    • Les variables personnelles qmake sont supportées
    • Un support pour les archives libtool, les fichiers pkgconf, les applications en terminal et les dlls de windows est ajouté
  • AMÉLIORÉ - Gestionnaire de projet automake : le support pour des cibles make arbitraires et les commandes dans le menu contextuel des sous-projets est ajouté
  • CORRIGÉ - Beaucoup de bogues dans le gestionnaire de projet Makefiles personalisées
  • AMÉLIORÉ - Support Valgrind : Valgrind 2.1 et supérieur sont maintenant supportés (des options de lignes de commandes correctes sont utilisées)
  • AMÉLIORÉ - intégration GUI Designer : KDevelop est maintenant compatible avec Kexi Formeditor
  • AMÉLIORÉ - KDevelop Designer :

    • Documentation settings dialog is visually AMÉLIORÉ
  • AMÉLIORÉ - Module externe KDevelop Assistant et documentation :
    • Les polices et les facteurs de zoom factors sont configurables
    • Les fichiers CHM sont affichés avec l'index
  • AMÉLIORÉ - Améliorations générales :
    • Réduction du temps de démarrage
    • Améliorations des propriétés de l'éditeur de bibliothèque
    • Les menus XDG sont créés
    • L'arborescence de source est restructurée et toutes les parties non utilisées sont supprimées
  • NOUVEAU - L'éditeur Kyzis (version CVS, www.yzis.org) fonctionne maintenant avec KDevelop
