Il comprend toutes les caractéristiques de KDevelop 3.1.x plus :
  • Code de base restructuré
    • Le code de base de KDevelop a été restructuré dans le but d'améliorer la réutilisation du code et de documenter les classes de la bibliothèque de base qui compose KDevelop. Ces classes de base forment la nouvelle plate-forme KDevelop et permettent une API bien documentée et concise.
  • Nouveaux outils RAD (Rapid Application Development : dévelopement rapide d'applications)
  • L'intégration de KDevDesigner permet la création des interfaces graphiques directement dans KDevelop
  • L'intégration de KDevAssistant fournit un visualiseur puissant de documentation dans KDevelop
  • L'assistant de classe C++ Class peut maintenant être utilisé pour les classes de la base de données de complétion de code. Cela signifie, par exemple, que si la base de données de complétion de code est faite pour Qt alors le surclassement des classes Qt est possible dans l'assistant.
  • Beaucoup d'amélioration du support du language ruby
    • Nouveau débogueur ruby, combiné avec des méthodes d'inspection améliorées dans les liens qtruby/korundum, pour montrer les metadata Qt::Object telles que les propriétés, pendant le débogage.
    • Le navigateur de classes a un analyseur amélioré qui montre les signals Qt et DCOP ainsi que les slots, les méthodes attr_accessor et les variables
    • Nouveaux modèles de projets pour une application KDE, une application KConfig XT, un serveur DCOP et une application Qt seulement
    • Intégration totale Qt designer/rbuic
  • Autres améliorations et corrections de bogues
    • Le support pour le contrôle de version a été refait. La plupart des caractéristiques d'intégration qui n'étaient pas disponibles dans KDevelop  3.1 sont de retour dans 3.2, avec un module externe Subversion amélioré.
    • L'amélioration de la recherche dans les fichiers, du support Ctags et de la navigation dans l'historique permettent une navigation bien plus rapide dans le code
    • Plus de modèles d'applications sont fournis
    • Surclassement des types bibliothèque
    • Veuillez voir la page du journal des changements pour plus de détails
