Il n'y a aucune raison de récriminer contre les bogues ou contre des caractéristiques manquantes. KDevelop peut seulement être bon si vous contribuez à l'améliorer.
Aussi nous avons besoin de vous ! Notre équipe recherche toujours de bons programmeurs Qt ou KDE.


Que faire pour devenir membre de l'équipe ?
  • avoir une idée de ce qui est ennuyant, mauvais ou manquant dans KDevelop et avoir la volonté de l'implémenter.
  • obtenir les sources de kdevelop SVN
  • tout compiler vous-même comme version deboguée pour être capable de tracer les crashes.
  • étudier le code source de kdevelop, déboguer KDevelop lui-même et étudier les backtraces pour en comprendre les mécanismes; reconnaître les classes C++ importantes et non-importantes du projet.
  • patcher les sources du projet
  • créer un fichier patch (*.diff) contre la version cvs actuelle
  • l'envoyer à kdevelop-devel at kdevelop.org
  • attendre la réponse, des cris et de l'amour
  • écrire d'autres patches aussi cools et enquiquiner la liste de diffusion avec eux à nouveau
  • connaître les règles de commit au SVN
  • obtenir votre propre compte kdevelop-cvs, et peut-être votre adresse votre nom at kdevelop.org pour pouvoir committer votre code vous-même
  • ...
