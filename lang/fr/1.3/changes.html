  • le nouveau Qt Linguist et ses outils l'accompagnant pour créer des fichiers d'internationalisation pour les applications Qt est complétement supporté. La manipulation d'internationalisation s'effectue de la même manière que pour le projet KDE avec KDevelop, sauf que vous devez utiliser tr() au lieu de i18(). Créez un exemple d'application Qt pour voir les détails. Sachez aussi que tous les utilitaires Qt doivent être dans votre variable $PATH, habituellement $QTDIR/bin
  • tous les modèles KDE 2.0 et Qt 2.2.x utilisent le système autoconf/automake de KDE, ce qui permet aux développeurs de convertir leur système de configuration automatique vers KDE facilement s'ils décident d'utiliser les fonctionnalités de KDE pour leur application.
  • KDevelop est livré avec trois scripts additionnels pour vous simplifier la vie; ils sont installés dans $prefix/bin (habituellement $KDEDIR/bin):
    • extractrc: c'est un script qui se trouve dans le module KDE CVS kdesdk (KDE Software Development Kit). Il est appelé automatiquement pour créer le fichier pot de votre application et extraire les chaînes de caractères de votre fichier de ressource XML qui est utilisé pour créer l'interface graphique (GUI) dans les applications KDE 2.0
    • kimport: kimport est un script perl qui vous permet de créer un fichier *.kdevprj (KDevelp Project File) à partir de presque tous les projets déjà compatibles automake/autoconf. Utilisation: allez dans le répertoire de votre projet et appelez kimport > <lenomdevotreprojet>.kdevprj. Ceci redirigera la sortie vers un fichier de projet que vous pouvez charger dans KDevelop
    • kdevdlg2ui: convertit un fichier *.kdevdlg existant qui a été créé avec l'éditeur de boîtes de dialogue de KDevelop vers un fichier *.ui qui peut être édité avec le concepteur de Qt
