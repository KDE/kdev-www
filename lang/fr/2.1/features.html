Gestion de projet :

Le fichier du projet garde toutes les informations sur les fichiers de votre projet telles les propriétés des fichiers (à inclure ou à exclure de la distribution) et des projets peuvent être créés et changés individuellement. Les projets générés sont compatibles avec autoconf/automake.

Éditeur de dialogue :

KDevelop vous fournit un moyen facile pour créer des interfaces graphiques (GUI) avec l'éditeur de dialogue intégré. Vous pouvez laisser KDevelop générer le code source du dialogue et obtenir le contrôle total de la fonctionnalité dialogue.

Parseur de classe / Outils de classe :

Le visualiseur de classe vous permet pour l'instant l'analyse de presque toutes les déclarations C++ et C, les classes emboîtées, les structures à l'intérieur des classes et les opérateurs ainsi que les espaces de noms. Il est aussi possible d'ajouter des méthodes et des attributs en utilisant les dialogues des outils de classes. L'outil de classe est une nouvelle caractéristique de KDevelop. L'idée est que vous voulez généralement un outil pour voir l'héritage et les méthodes qui peuvent être utilisées par héritage. Ce besoin est comblé par l'outil de classe. Il offre la navigation dans les classes du système d'une façon intuitive et permet de visualiser les relations entre elles.

Débogueur intégré :

Avec KDevelop 1.1 est fourni un débogueur intégré complet qui vous permet d'utiliser le visualiseur de classe de KDevelop de façon encore plus efficace quand vous déboguez car vous accédez facilement au code source pour inclure des points d'arrêt et observer les variables de votre application.

Le visualiseur de classe graphique :

Le visualiseur de classe graphique vous offre la possibilité d'obtenir une vue générale de votre projet et de toutes vos classes.

Le créateur d'applications :

Le créateur d'applications (KAppWizard) offre la possibilité de générer différents squelettes d'applications pour créer de nouveaux programmes. Sont disponibles une application KDE standard avec barre de menus, barre d'outils et barre d'état, une mini-application KDE avec une fenêtre principale "Mainwindow" vide, une application GNOME complète, une application basée seulement sur Qt avec barres de menus, d'outils et d'état, et finalement une application-type Terminal C/C++. KDevelop 1.1 inclut même des squelettes KDE 2/Qt 2 pour les développeurs qui veulent rester à la pointe du développement KDE. Toutes ces applications sont générées et adaptées aux besoins de votre nouvelle application et sont prêtes à s'exécuter en quelques secondes.

Documentation :

KDevelop est accompagné d'une documentation complète qui consiste en cinq guides : le manuel de l'utilisateur, le manuel de programmation, le tutoriel et les guides de référence des bibliothèques KDE. Cette documentation à laquelle s'ajoutent une référence C/C++ et la documentation QT et KDE peut facilement être consultée avec le navigateur de documentation intégré.

Quatre fenêtres :

Les fenêtres vous permettent de changer facilement entre les fenêtres des fichiers sources/en-têtes, des classes et de la documentation ce qui vous laisse gérer les fichiers de votre projet sans avoir à utiliser un gestionnaire de fichiers externe.

Le navigateur de documentation vous offre un accès direct à la documentation actuelle des bibliothèques Qt et KDE, aux manuels et tutoriels KDevelop. Avec l'aide de l'arborescence de documentation, vous pouvez accéder directement à la documentation pour toute classe ou méthode KDE/Qt.

Le système d'aide fournit des messages d'aide rapide,"Qu'est-ce que c'est"- et l'aide de la barre d'état partout, vous bénéficiez toujours d'aide sur ce que vous pouvez faire et où vous êtes.

3 modes de visualisation MDI dans un seul choix :

Transformons l'interface graphique utilisateur de KDevelop pour qu'elle ressemble à celle de MS Visual C++, Borland Builder, KDEStudio ou celle du Gimp. Amarrez les fenêtres de vue comme vous le voulez.Cette interface vous permet :

  • d'ouvrir autant de fichiers source que vous le voulez
  • d'ouvrir autant de fichiers de navigation dans la documentation que vous le voulez
  • d'ouvrir des vues multiples pour un fichier source.
  • de travailler dans le mode interface utilisateur de votre choix. Vous pouvez basculer entre les choix suivants :
    1. Chaque vue est une fenêtre principale (toplevel) sur le bureau. Même les vues outils (arborescences et sortie). En plus, il y a une fenêtre de contrôle contenant les barres et les menus. Comme dans le Gimp.
    2. Une "childframe" MDI comme dans MS Visual C++.
    3. Un mode TabPage MDI comme dans KDEStudio ou Borland Builder.
  • d'utiliser une barre des tâches propre à l'application pour les vues ouvertes (mais bien sûr, vous pouvez ne pas utiliser cette option)
  • travailler avec des vues des outils totalement flottantes


Ceci est l'interface utilisateur par défaut, un scenario qui ressemble pas mal à MSVC ;-). Notez la vue de la barre des tâches en bas. Et, de plus, est-ce que vous voyez le petit bouton "undock" dans le coin supérieur droit de la fenêtre (à côté du bouton Réduire de la vue de l'éditeur) ? Si vous cliquez dessus, la vue de l'éditeur va se séparer du bureau et devenir une fenêtre principale.
Ceci est le mode TabPage MDI. Il semble d'après les statistiques que c'est le mode utilisé le plus souvent.
Je suppose que la plupart des utilisateurs Unix sont habitués à travailler en mode toplevel où chaque vue est directement sur le bureau. Rien de plus facile avec KDevelop-2.0! Vous pouvez même arracher les outils de visualisation en dehors de leur fenêtre. Par exemple, vous voyez que j'ai tiré le navigateur de documentation hors de l'arborescence. Note : rattacher une des vues à la fenêtre principale signifie revenir au mode "Childframe".
