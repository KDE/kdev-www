Cette page rassemble toutes les informations sur les séries KDevelop 2.x

Le développement a été arrêté le 19 Novembre 2002 avec la sortie de KDevelop 2.1.4 pour KDE 3.0, et aucune autre sortie de version 2.x n'est planifiée. KDevelop 2.1.5 fut une exception, sortie le 28 Janvier 2003, en tant que version de mise à jour pour les utilisateurs de KDE 3.1. Le développement se fait maintenant sur le code du nouveau KDevelop 3.0. Plus de 80 pour cent des bogues rapportées dans KDevlop 2.x sont résolues dans KDevelop 3.0.

Compiler la version 2.1.5 du CVS

Pour obtenir un KDevelop 2.1.5 compilé depuis les sources cvs, faire un checkout du module CVS "kdevelop" à :pserver:anonymous@anoncvs.kde.org:/home/kde en utilisant la branche CVS de corrections des bogues KDE_2_2_BRANCH
Vous devez aussi faire un checkout du module kde-common depuis cvs.kde.org pour avoir le répertoire admin si vous avez KDE 3.
Veuillez faire attention aux différentes branches cvs que certaines versions de KDE 3 nécessite :
  • KDE-3.0.x nécessite le marqueur KDE_3_0_BRANCH,
  • KDE-3.1.x nécessite le marqueur KDE_3_1_BRANCH,
  • seul cvs HEAD ne nécessite pas de marqueur cvs pour kde-common,
C'est pour cela que dans ce cas vous n'avez pas besoin d'utiliser l'option '-r nomMarqueur_ciDessus'.

Ligne de commande si vous avez KDE 3 :
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r nomMarqueur_ciDessus kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

Ligne de commande si vous avez KDE 2.2 :
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Pour compiler et installer KDevelop (version CVS) sur votre système,
veuillez taper les commandes ci-dessous dans le répertoire de base de votre distribution KDevelop :

une version compilée : 
       $ export KDEDIR=/où/est/votre/kde2/ou/kde3/
       $ export QTDIR=/où/est/votre/qt-2.3.1/ou/qt-3.0.x/
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (en tant que root)

une version déboguée-compilée : 
       $ export KDEDIR=/où/est/votre/kde2/ou/kde3 
       $ export QTDIR=/où/est/votre/qt-2.3.1/ou/qt-3.0.x
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (en tant que root)
Note : Pour KDE 3 vous avez besoin au moins de Qt > 3.0.3 (le mieux est qt-copy de KDE), pour KDE 2.2.x vous avez besoin de Qt 2.3.x
Veuillez éviter de lier les deux versions Qt, Qt2 et Qt3 ! Sinon le démarrage de KDevelop va échouer avec un crash mystérieux...
