  • L'interface utilisateur de KDevelopa été re-écrire avec l'utilisation de l'architecture d'interface de plusieurs documents, permettant de visualiser plusieurs fenêtres. Plusieurs visualisations du même fichier peuvent être utilisées.
  • Le mode de visualisation peut être basculé entre le mode Childframe, le mode Toplevel et le mode TabPage.
  • La gestion des signets et le débogueur interne ont été adaptés à la nouvelle interface.
  • Ces deux caractéristiques sont possibles car nous sommes basés maintenant sur QextMDI, une bibliothèque qui supporte le MDI avancé pour Qt et KDE.
  • Un nouvel assistant de configuration (basé sur QWizard) remplace l'ancien, basé sur une messagebox.
  • Des sessions de projet (.kdevses) ont été introduites.
  • L'utilisation du Dockwidget a été etendue pour amarrer/désamarrer les fenêtres de tabulation dans les arborescences de vue et la visualisation des sorties.
  • De nouveaux squelettes pour implanter une bibliothèque de style KDE/Qt et des modules de KControl modules ont été ajoutés ainsi que des squlettes pour des modules externes de Konqueror et des applications Kicker.
  • La copie du répertoire kde-common/admin copy (admin.tar.gz) a été mise à jour vers la dernière version de KDE CVS. Les utilisateurs de versions antérieures à 2.0 sont avisés qu'ils doivent mettre à jour le répertoire admin dans leur projet pour le support des nouvelles versions de autoconf et automake.
  • KDevelop utilise maintenant le dialogue de l'astuce du jour de kdelibs.
  • Le manuel utilisateur a été étendu pour décrire la nouvelle interface utilisateur et un chapitre a été ajouté pour utiliser Qt Designer avec des projets KDevelop.
  • Une base de données nouvellement ajoutée basée sur ctags permet "Goto definition/declaration" et la navigation entre les fichiers .h et .cpp. Cela nécessite ctags-5.0.1.
  • La sortie du compilateur s'affiche en couleur.
  • Les pages sont accessibles via le navigateur d'aide de KDevelop.
  • La fonctionalité de KDevelop a été séparée en bibliothèques partagées.
  • Les sons pour les évènements de KDevelop (les notifications système dans le Centre de Contrôle de KDE) sont supportés.
  • La configuration de sous-projets de bibliothèques partagées est améliorée.
  • Beaucoup de corrections de bogues, grosses ou petites, sont faites, en particulier dans l'éditeur.
  • Le débogueur interne de KDevelop reconnaît maintenant les chaînes de Qt-3.0.
  • Les fichiers .po et .xpm peuvent être ouverts comme fichiers ASCII.
