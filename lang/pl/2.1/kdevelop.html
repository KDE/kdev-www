Ta strona zawiera wszystkie informacjie o wydaniach KDevelop 2.x

Rozwijanie zatrzmano w dniu 19. Listopada 2002 wraz z wersją KDevelop 2.1.4 dla KDE 3.0, nie planuje się kolejnych wydań z serii 2.x. KDevelop 2.1.5 był wyjątkiem, wydany 28. Stycznia 2003, jako aktualizacja dla użytkowników KDE 3.1. Zaczęto rozwijać nowszy kod KDevelop 3.0. Więcej niż 80% zgłoszonych błędów w KDevlop 2.x są poprawione w KDevelop 3.0.

Kompilacja wersji 2.1.5 z repozytorium CVS

Aby pobrać własnoręcznie-skompilowane KDevelop 2.1.5 ze źródeł cvs, sprawdź moduł CVS "kdevelop" na :pserver:anonymous@anoncvs.kde.org:/home/kde używając wolnej od błędów gałąź CVS - KDE_2_2_BRANCH
Będziesz również potrzebować kde-common z cvs.kde.org aby dostać się do katalogu głównego jeśli korzystasz z KDE 3.
Zwróć uwagę, że różne gałęzie cvs wymagają odpowiednich wersji KDE 3:
  • KDE-3.0.x wymaga etykiety KDE_3_0_BRANCH,
  • KDE-3.1.x wymaga etykiety KDE_3_1_BRANCH,
  • tylko HEAD nie potrzebuje etykiety cvs dla kde-common,
to dlatego nie należy używać opcji '-r nazwaEtykiety_powyżej'.

jeśli używasz KDE 3:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r tagName_seeAbove kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

jeśli używasz KDE 2.2:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Aby skompilować i zainstalować KDevelop (wersja-CVS) na twoim systemie,
wpisz poniższe komendy w katalogu głównym KDevelop:

wersja kompilowana pod wydanie: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

wersja kompilowana pod debugowanie: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)
Uwaga: Dla KDE 3 potrzebujesz przynajmniej Qt > 3.0.3 (lepiej qt-copy KDE ), dla KDE 2.2.x potrzebujesz Qt 2.3.x
Unikaj linkowania obu wersji Qt, Qt2 i Qt3! W innym przypadku start KDevelop nie powiedzie się z wieloma tajemniczymi raportami...

Pamiętaj, że to są łatki na źródła, więc musisz pobrać kompatybilne źródła łatek, załatać i przekompilować.
