Для створення звіту чи побажання, потрібно відвідати мастер складання звітв про помилки нашої бази даннихbugzilla. Ви не повинні репортувати про помилки при компіляції в базу данних багів. Для помилок при компіляції використовуйте mailing list.

Для повної доповіді на KDevelop's про поточні баги чи побажання запит до бази данних багів. Доповідь поділисться на компоненти, отож потрібно лише перейти на Report Count by Component and Severity.

Для більш докладного перегляду багів, активуйте cookies у вашому браузері та перейдіть bugzilla column configuration page and select:
  • changeddate
  • severity
  • priority
  • reporter
  • status
  • component
  • version
  • votes
  • Summary (first 60 characters)

Це все, тепер всі запити на баги будуть мати деталі потрібні для KDevelop.
