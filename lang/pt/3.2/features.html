Contém todas as funcionalidades do KDevelop 3.1.x mais:
  • Nova base (reestructurada) do código
    • A base do código do KDevelop foi reestructurada com o objectivo de melhorar a reutilização do código e documentar as bibliotecas base que compõem o KDevelop. Estas classes base formam a nova KDevelop platform e apresentão uma concisa e bem documentada API
  • Novas ferramentas RAD (Rapid Application Development)
    • Integração com o KDevDesigner posiblita o design de GUI dentro do próprio KDevelop
    • Integração com o KDevAssistent possibilita a consulta de documentação dentro do próprio KDevelop
    • O C++ Class Wizard agora pode ser usado para classes do repositório permanente de classes. Isto significa, por examplo, que se a base de dados Code Completion para o Qt fôr construida então derivar subclasses das classes do Qt é possível usando o Wizard
  • Muitas melhorias no suporte da linguagem ruby
    • Novo depurador ruby, combinado com métodos aperfeiçoados de inspeção baseados no qtruby/korundum, para mostrar metadados do Qt::Object tais como propriedades, enquanto depura
    • O navegador de classes tem um parser aperfeiçoado que mostra sinais e slots Qt e DCOP, métodos attr_accessor e instâncias variáveis
    • Novos modelos de projeto para aplicações KDE, KConfig XT, servidor DCOP e Qt simples
    • Completa integração com Qt designer/rbuic
  • Outras melhorias e reparos de bugs
    • O suporte ao controle de versões ( Version Control ) foi refeito. A maior parte das características de integração que estavam desabilitadas no KDevelop 3.1 estão no 3.2, junto com um plugin Subversion muito melhorado
    • Função "Find in files" aperfeiçoada, suporte para Ctags e história de navegação ajudando a uma navegação mais rápida pelo código
    • Mais modelos de aplicações
    • Novos modelos para cmake baseados em aplicações C/C++ e bibliotecas
    • Novos modelos para aplicações KDE baseadas em scons
    • Veja a página change log para mais detalhes
