Não existem razões para queixas sobre bugs e inexistência de funcionalidades. O KDevelop pode apenas ser tão bom tanto quanto o que contribuíres para o melhorares.
Logo precisamos de ti! A nossa equipa procura sempre bons programadores de QT ou KDE.


Então o que é preciso para me tornar um membro da equipa?:
  • ter uma ideia do que é aborrecido, mau ou em falta no KDevelop e ter a força de vontade para o implementar.
  • obter o código fonte do KDevelop do SVN
  • compilar tudo como versão de debug para localizar os crashes
  • estudar o código do KDevelop, efectuar o debug do próprio KDevelop e analizar os backtraces para perceber os vários mecanismos; reconhecer as classes importantes e menos importantes do projecto
  • corrigir as fontes actuais
  • criar um ficheiro patch (*.diff) relativo á versão actual de cvs
  • enviá-lo para kdevelop-devel at kdevelop.org
  • obter resposta, criticas e amor
  • escrever muitas mais correções porreiras e chatear a mailing-list com estas
  • aprender as regras de commit do SVN
  • obter a tua própria conta kdevelop-cvs, e talvez mesmo obter o teu próprio endereço em kdevelop.org para te permitir adicionar o teu código por ti mesmo
  • ...
