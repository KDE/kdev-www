1. Antes de continuar verifique por favor se o seu computador tem o software necessário instalado

  2. Obtenha as fontes como é explicado acima.

  3. Inicialize o sistema de compilação:
    $ cd kdevelop
    $ make -f Makefile.cvs
Após ter descarregado e compilado com sucesso um dos ramos SVN acima, irá querer mantê-lo actualizado para obter todas as correcções.
  1. De forma a evitar escrever todos os comandos que definem as necessárias variáveis de ambiente sempre que actualiza, é aconselhável defini-las no seu ficheiro .bashrc ou .cshrc.
    • Se utiliza o interpretador de comandos bash adicione as seguintes linhas:
      export KDEDIR=/onde/está/o/seu/kde3/
      export QTDIR=/onde/está/o/seu/qt3/
      export KDEVELOPDIR=/where/kdevelop/will/be/installed
      export KDEDIRS=$KDEVELOPDIR:$KDEDIR
      export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      export PATH=$KDEVELOPDIR/bin:$QTDIR/bin:$KDEDIR/bin:$PATH
    • Se usa tcsh adicione as seguintes linhas:
      setenv KDEDIR /onde/está/o/seu/kde3/
      setenv QTDIR /onde/está/o/seu/qt3/
      setenv KDEVELOPDIR /where/kdevelop/will/be/installed
      setenv KDEDIRS $KDEVELOPDIR:$KDEDIR
      setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      setenv PATH $KDEVELOPDIR/bin:$QTDIR/bin:$KDEDIR/bin:$PATH
    Nota: Por razões obvias deve editar as três primeiras linhas de acordo com a sua configuração! :)


  2. Na directoria onde colocou os ficheiros do KDevelop do SVN execute o seguinte comando:
    $ cd kdevelop
    $ svn up


  3. O próximo passo depende do resultado do comando anterior:
    • Se obteve algo parecido com isto:
      U /some_directory_path/Makefile.am
      Deve fazer o seguinte:
      $ make -f Makefile.cvs
      seguido pelo seguinte passo de ./configure


    • Se obteve algo parecido com isto:
      P /some_directory_path/Makefile.am
      Deve fazer:
      $ ./configure --enable-debug=full --prefix=$KDEVELOPDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
      usando as opções de ./configure que usou quando compilou o KDevelop pela primeira vez.


  4. Compile o KDevelop e instale-o:
    $ make
    $ make install (como root)
    Se por alguma razão o comando make falhou pode obrigá-lo a ignorar os erros utilizando a opção -k desta forma:
    $ make -k
    $ make -k install (como root)
    Dependendo da gravidade do erro pode acabar com uma versão a trabalhar ou não do KDevelop. Por isso ao executar "make -k install" está por sua própria conta e risco!!!. De qualquer forma estes erros normalmente desaparecem ao fim de algumas horas por isso pode tentar um "update" mais tarde antes do perigoso :) comando make install.


E é tudo! Você tem agora a versão mais actualizada do KDevelop. Para o iniciar só tem que fazer:
$ kdevelop

Tudo o que tem que fazer para o manter actualizado é repetir estes passos sempre que alguém na lista de discussão kdevelop-devel corriga um bug. Por isso é fortemente aconselhado a se juntar a esta lista de discussão.

Você deve reportar os bugs que encontra no formulário de submissão de bugs. Você não deve reportar erros de compilação na base de dados de bugs. Para erros de compilação use antes a lista de discussão.

Pode consultar os bugs na base de dados de bugs.
