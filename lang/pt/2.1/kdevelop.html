Esta página concentra toda a informação sobre a série KDevelop 2.x

O desenvolvimento terminou em 19 de Novembro de 2002 com o lançamento do KDevelop 2.1.4 para o KDE 3.0, e não se encontram planeadas novas versões 2.x. O KDevelop 2.1.5 foi uma excepção, sendo lançado em 28 de Janeiro de 2003, como uma versão de actualização para os utilizadores de KDE 3.1. O desenvolvimento mudou para o novo código base do KDevelop 3.0. mais de 80% dos bugs reportados no KDevlop 2.x estão corrigidos no KDevelop 3.0.

Compilação da versão 2.1.5 a partir do repositório CVS

Para obter automaticamente o KDevelop 2.1.5 compilado a partir das fontes em cvs, faça o checkout do módulo CVS "kdevelop" em :pserver:anonymous@anoncvs.kde.org:/home/kde utilizando o ramo CVS corrigido KDE_2_2_BRANCH
Terá também de efectuar o checkout do kde-common de cvs.kde.org para obter o directório admin se está a utilizar KDE 3.
Preste atenção aos diferentes ramos cvs necessários para certas versões do KDE 3:
  • KDE-3.0.x necessita da tag KDE_3_0_BRANCH,
  • KDE-3.1.x necessita da tag KDE_3_1_BRANCH,
  • apenas o HEAD cvs não necessita da tag cvs para o kde-common,
é por essa razão que não deve usar a opção '-r nome_da_tag_ver_acima'.

linha de comandos se está a correr KDE 3:
       $ mkdir kde3src
       $ cd kde3src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r nome_da_tag_ver_acima kde-common
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop
       $ rm -rf admin
       $ ln -s ../kde-common/admin admin

linha de comandos se está a correr KDE 2.2:
       $ mkdir kde2src
       $ cd kde2src
       $ cvs -z3 -d :pserver:anonymous@anoncvs.kde.org:/home/kde co -r KDE_2_2_BRANCH kdevelop
       $ cd kdevelop

Em ordem a compilar e a instalar o KDevelop (versão CVS) no seu sistema,
escreva os comandos abaixo na directoria base da distribuição KDevelop:

para uma versão de lançamento: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --prefix=$KDEDIR
       $ make
       $ make install (as root)

para uma versão de debug: 
       $ export KDEDIR=/where/your/kde2/or/kde3/is
       $ export QTDIR=/where/qt-2.3.1/or/qt-3.0.x/is
       $ export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
       $ export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
       $ export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH

       $ export CXXFLAGS="-g3 -O0"

       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs
       $ ./configure --enable-debug --prefix=$KDEDIR
       $ make
       $ make install (as root)
Nota: Para KDE 3 necessita pelo menos do Qt > 3.0.3 (a melhor será a qt-copy do KDE), para KDE 2.2.x precisa do Qt 2.3.x
Evite a linkagem com ambas as versões Qt, Qt2 e Qt3! Caso contrário o arranque do KDevelop irá falhar com um misterioso relatório de estoiro...

Plugins e correcções

Devido a várias razões (principalmente a estabilidade), algumas correcções maiores não podem ser aplicadas na versão currente. Estas no entanto oferecem grandes melhoramentos, por isso podem ser descarregadas aqui.
Lembre-se que se tratam de correcções para o código, logo terá que descarregar uma versão do código compatível, aplicar as correções e recompilar.

As construções não são feitas nas sub-directorias apropriadas, as configurações de construção não podem removidas e o debugging sobre estas não funciona correctamente. Esta correcção corrige estes problemas. Eu estou a manter e continuo a desenvolver uma grande quantidade de projectos para processadores ix86 e arm. O trabalho utilizando esta correcção tem corrido bem à já pelo menos meio ano. A configuração começa com Opções->Configuração do KDevelop->Compilador onde devem ser definidos os caminhos apropriados para os compiladores. Eu devo dizer que prefiro as configurações de construção do KDevelop2 às do KDevelop3 dado que o primeiro guarda as informações do compilador separadas das do projecto, e possíveis de alternar através do switch de host.

separação dos recursos de dados dos do KDevelop3		 Iztok (I (in) siol .. net) 2.1 e versões de correcção de bugs (2.1.x), mas apenas para KDE-3 Como o Gideon (KDevelop3) foi renomeado para KDevelop, os seus recursos de dados têm chocado com os do KDevelop2. Esta correcção resolve o seguinte:
  • o binário kdevelop é renomeado para kdevelop2
  • os ficheiros kdevelop2.desktop e kdevelop2-setup.desktop são adicionados á directoria kevelop/
  • as directorias $KDEDIR/share/..../kdevelop são renomeadas para kdevelop2
  • a directoria $HOME/.kde/share/..../kdevelop é renomeada para kdevelop2

Bugs conhecidos
