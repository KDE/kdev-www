Чтобы добавить отчет об ошибке или пожелание, используйте мастер сообщения об ошибках в нашей базе данных bugzilla. Не следует сообщать о проблемах компиляции в базу данных ошибок. Для таких случаев следует использовать список рассылки.

Список ошибок и пожеланий для KDevelop доступен в базе данных ошибок. Отчеты разделены по компонентам, так что возможно получить Отчет о компонентах.

Для детального просмотра ошибок следует разрешить "cookie" в обозревателе, перейти на эту страницу и выделить:
  • changeddate
  • severity
  • priority
  • reporter
  • status
  • component
  • version
  • votes
  • Summary (первые 60 символов)

Вот и все, теперь запросы на получение списка ошибок будут максимально информативны.
Удачной охоты за ошибками! :)
