Содержит возможности KDevelop 3.1.x плюс:
  • Новый переработанный код
    • Исходный код KDevelop был переработан с целью улучшить повторное использование кода и документирования классов базовой библиотеки, которые составляют KDevelop. Базовые классы образуют новую платформу KDevelop и представляют краткий и хорошо документированный API
  • Новые RAD (Rapid Application Development) инструменты
    • Интеграция с KDevDesigner позволяет редактировать GUI не выходя из KDevelop
    • Интеграция с KDevAssistent позволяет просматривать документацию не выходя из KDevelop
    • Мастер классов C++ теперь может быть использован для классов из постоянного хранилища классов. То есть, например, если база данных авто-дополнения кода для Qt уже построена, тогда в мастере возможно создание подклассов от классов Qt
  • Значительно улучшена поддержка ruby
    • Новый отладчик ruby, совмещенный с улучшенными инспектирующими методами в qtruby/korundum, для просмотра метаданных Qt::Object в виде своиств, во время отладки
    • В модуле просмотра классов улучшен синтаксический анализатор, который показывает Qt и DCOP сигналы и слоты, attr_accessor методы и instance переменные
    • Новые шаблоны проектов для приложений KDE, KConfig XT, DCOP сервера и просто Qt приложений
    • Полная интеграция Qt designer/rbuic
  • Другие улучшения и исправления
    • Переработана поддержка систем управления версиями. Большинство опций интеграции которые были заблокированны в KDevelop 3.1 снова доступны в 3.2, вместе с улучшенным модулем Subversion.
    • Улучшен поиск "Find in files", поддержка Ctags и навигация по истории позволяют быстро перемещаться по исходному коду
    • Добавлены новые шаблоны проектов
    • Новые шаблоны для основанных на cmake приложений и библиотек C/C++
    • Новые шаблоны для основанных на scons KDE приложений
    • Смотрите более детальный список изменений
