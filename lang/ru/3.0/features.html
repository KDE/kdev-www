• Модульная архитектура
    • Функциональность может быть добавлена, изменена или удалена без внесения изменений в ядро программы.
    • Лицензия библиотек (LGPL) позволяет создание коммерческих и свободных модулей IDE.
    • Имеется поддержка профилей, определяющих набор модулей для загрузки, что особо применимо для специфических окружений.
  • Управление проектом
    • Полная поддержка automake и qmake. Изменения в проект внесенные вручную позволяются и распознаются интегрированной средой. Также все существующие проекты могут быть легко импортированы.
    • Возможность создания собственных файлов makefile для сборки проекта.
    • Поддержка ant - средства сборки проектов на языке java.
    • Поддержка дополнительных типов проектов для языков pascal, haskell, ada а также интерпретируемых языков.
    • Инфраструктура для добавления поддержки любых систем сборки с помощью создания модулей обобщенного менеджера проектов.
    • Мастер создания файлов для управления шаблонами файлов и создания из них файлов в проектах.
    • Модули компилятора для указания параметров компилятора в удобных диалогах.
    • Профили конфигурации с возможностью указания различных компиляторов, флагов компиляции, каталогов сборки, целевой архитектуры и т.д.
    • Поддержка средств документирования проекта таких как doxygen для автоматического создания документации по API проекта (в формате html, tex и man) с перекрестными ссылками на другие используемые библиотеки.
    • Поддержка интернационализации, позволяющая переводчикам легко добавлять языки в проект, включая поддержку KBabel.
    • Поддержка проектов Qt/Embedded (для платформ Zaurus и iPAQ).
  • Поддержка языков программирования
    • Разработка возможна на 15 языках включая Ada, C, C++, Objective-C, SQL, Fortran, Haskell, Java, PHP, Pascal, Perl, Python, Ruby, Bash, XUL (неофициально).
    • Просмотр проблем дающий информацию о различных "проблемах" в исходном коде по мере набора. Проблемами могут быть ошибки синтаксиса, записи TODO и FIXME.
    • Просмотр классов отображающий иерархию классов и других символов в проекте.
    • Средства грамматического разбора языков для наполнения средств просмотра классов и проблем.
    • Мастера создания классов, атрибутов и методов для объектно-ориентированных языков.
    • Автоматическое дополнение кода и подсказка по коду для переменных и методов классов, аргументов функций и других символов.
    • Форматирование исходного кода, подсветка синтаксиса и сворачивание блоков кода.
    • Специфические для языка C++ возможности такие как:
      • "сгенерировать член класса" - создание определение метода в исходном файле на основании определения метода в заголовочном файле;
      • "извлечь интерфейс"- извлечение интерфейса (объявлений методов) и создание абстрактного базового класса с этим интерфейсом;
      • "создание производных классов" - автоматизированное создание производных от форм Qt Designer классов с одновременным переопределением слотов.
  • Интегрированный отладчик
    • Интерфейсы к GDB и Java debugger.
    • Интегрированная консоль GDB.
    • Точки останова на строке, по адресу и в методе в также точки наблюдения.
    • Условные точки останова.
    • Дизассемблер и просмотр стека вызовов.
    • Дерево наблюдения переменных.
    • Просмотр памяти, регистров и загруженных библиотек.
    • Поддержка отладочных оболочек (libtool).
    • Удаленная отладка.
  • Редактирование кода
    • Любой редактор, соответствующий интерфейсу KDE KTextEditor может быть использован с KDevelop.
    • В настоящее время доступны редакторы Kate, QEditor (основан на редакторе Qt Designer) и KVim.
    • Хранимые в проекте списки избранных строк кода.
    • Просмотр различий между текстом в редакторе и на диске.
    • Просмотр различий для файлов исправлений.
    • Интеграция Grep - поиск по регулярным выражениям в проекте.
    • Средство поиска и замены по проекту.
  • Система контроля версий
    • Единое API для включения поддержки систем контроля версий в KDevelop.
    • Поддерживаемые системы: CVS, Perforce, Subversion и Clearcase.
  • Мастер создания приложений
    • Мастер создания приложений создает полноценные, готовые к использованию приложения.
    • Существуют шаблоны для каждого из поддерживаемых языков и систем сборки.
    • Возможность использования собственных шаблонов приложений.
    • Множество шаблонов проектов для KDE (приложения KPart, простые приложения KDE, модули KControl, апплеты Kicker (панели), приложения KOffice, KIOSlave, модули Konqueror, Noatun и Kate, стили рабочего стола).
  • Просмотр документации
    • Унифицированный интерфейс к просмотру документации.
    • Просмотр содержания, индекса и полнотекстовый поиск (с помощью htdig).
    • Автоопределение и включение документации по библиотекам Qt и KDE, а также документации в формате DevHelp.
    • Поддержка собственных коллекций документации (в формате KDevelopTOC).
    • Способность открывать локальную, удаленную и даже сжатую документацию (используя любой протокол, поддерживаемый Konqueror - http, ftp, sftp, tar, zip, и т.д.).
    • Способность открывать документацию в различных форматах (html, text, ps, pdf, chm, и т.д.).
  • Другие возможности
    • Пользовательский интерфейс полностью настраиваем. Возможно переключение на лету между классическими режимами MDI (плавающие окна, дочерние окна) и новым режимом IDEAl использующим отображения.
    • Встроенная консоль.
    • Выбор файлов, дерево файлов и средство группирования файлов.
    • Включение любой необходимой программы путем добавления в меню инструментов.
    • Проверка регулярных выражений.
    • Интеграция Valgrind для нахождения утечек памяти в программах.
    • Средство публикации и растпространения проектов для автоматического создания пакетов с исходниками и бинарниками.
    • Интеграция CTags.
    • Part explorer для опроса установленных сервисов KDE.
