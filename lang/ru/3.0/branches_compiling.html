Получите исходный код ветки KDE_3_2_BRANCH как показано выше.

Перед тем как продолжать, необходимо убедиться в наличии необходимого ПО.
Initialize the build system: 
       $ cd kdevelop
       $ make -f admin/Makefile.common cvs-clean
       $ make -f Makefile.cvs

После загрузки и компиляции одной из перечисленных выше веток CVS возможно проводить дальнейшие обновления. Для уменьшения количества команд для набора рекомендуется установить необходимые переменные окружения в файлах .bashrc или .cshrc.
При использовании оболочки bash необходимо добавить следующие команды:
export KDEDIR=/путь/к/kde3
export QTDIR=/путь/к/qt3
export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH
При использовании оболочки tcsh необходимо добавить следующие команды:
setenv KDEDIR /where/your/kde3/is
setenv QTDIR /where/your/qt3/is
setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
setenv PATH $QTDIR/bin:$KDEDIR/bin:$PATH

Заметка: По очевидным причинам необходимо отредактировать первые две строки в соответствии с имеющейся конфигурацией! :)


Далее необходимо перейти в каталог с кодом kdevelop из CVS и выполнить команду обновления:
 $ cd путь_к_kde3src/kdevelop
 $ cvs update -dP

Следующий шаг зависит от вывода предыдущей команды:
Если в выводе присутствуют строки вида:
           U /путь/Makefile.am
то необходимо выполнить:
 $ make -f Makefile.cvs
и далее _ВСЕ_ следующие шаги.

Если имеются:
           P /путь/Makefile.am
то:
 $ ./configure --enable-debug=full --prefix=$KDEDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
и далее _ВСЕ_ следующие шаги.

  $ make
  $ make install (с правами root)
Если по какой-либо причине команда make завершается неудачно, то можно игнорировать ошибки указывая параметр -k. В зависимости от степени тяжести ошибки можно получить как работающую так и неработающую версию kdevelop. Так что, используйте "make -k install" на собственное усмотрение!!!. В любом случае такие ошибки обычно уходят через пару часов, так что можно попытаться обновиться попозже перед страшной :) командой make install.
 $ make -k
 $ make -k install (с правами root)
Вот и все! Теперь должна быть установлена последняя версия KDevelop. Для запуска выполнить:
 $ kdevelop
Все, что необходимо делать для поддержки свежей версии - это повторять предыдущие шаги в случае если кто-либо в списке рассылки kdevelop-devel@kdevelop.org исправит ошибку.
Так что приглашаем подписаться на список рассылки.


Ошибки следует сообщать с помощью формы сообщений об ошибках. Не следует сообщать там об ошибках компиляции. Для таких случаев необходимо использовать список рассылки.

Взглянуть на ошибки можно в базе данных об ошибках.

Информация о последних изменениях в CVS доступна на странице архива списка kde-cvs.

Следуйте обычной процедуре, заменяя make на gmake и после 
       $ gmake -f Makefile.cvs
выполните: 
       $ export CXXFLAGS="-L/usr/X11R6/lib -I/usr/local/include/db41 -I/usr/local/include -L/usr/local/lib -I/usr/X11R6/include"
Необходимо иметь установленню версию berkeley db 4.1, при наличии другой версии следует подправить пути.
Измените строку "ac_cv_bdb=no" на "ac_cv_bdb=yes" в файле configure, в противном случае configure не найдет библиотеку db.
Для просмотра информации об установленной версии : 
       $ pkg_info | grep db41
       db41-4.1.25_1       The Berkeley DB package, revision 4.1
Разработчики FreeBSD переименовали libdb.so в libdbxx.so. Замените "-ldb" на "-ldb41" в lib/catalog/Makefile.am
Далее как обычно: 
       $ ./configure --prefix=/путь_установки --enable-debug=full
