1. Перед тем как продолжать, необходимо убедиться в наличии необходимого ПО.
  2. Получите исходники ветки KDE_3_3_BRANCH согласно вышеприведенной информации.

  3. Инициализируйте систему сборки:
    $ cd kdevelop
    $ make -f admin/Makefile.common cvs-clean
    $ make -f Makefile.cvs
После успешной загрузки и компиляции ветки CVS, можно поддерживать ее в обновленном состоянии чтобы всегда иметь возможность использовать последние исправления.
  1. Вместо набора инструкций можно устанавливать переменные окружения в файлах .bashrc или .cshrc.
    • При использовании оболочки bash:
      export KDEDIR=/путь/к/kde3
      export QTDIR=/путь/к/qt3
      export LD_LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      export LIBRARY_PATH=$QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      export PATH=$QTDIR/bin:$KDEDIR/bin:$PATH
    • при использовании оболочки tcsh:
      setenv KDEDIR /путь/к/kde3
      setenv QTDIR /путь/к/qt3
      setenv LD_LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LD_LIBRARY_PATH
      setenv LIBRARY_PATH $QTDIR/lib:$KDEDIR/lib:$LIBRARY_PATH
      setenv PATH $QTDIR/bin:$KDEDIR/bin:$PATH
    Заметка: По очевидным причинам необходимо отредактировать первые две строки в соответствии с имеющейся конфигурацией! :)


  2. Перейдите в каталог с кодом kdevelop из CVS и выполните команды:
    $ cd kdevelop
    $ cvs update -dP


  3. Следующий шаг зависит от вывода предыдущей команды:
    • Если вы получили что-то вроде:
      U /путь/Makefile.am
      необходимо выполнить:
      $ make -f Makefile.cvs
      за которым должен следовать ./configure


    • Если получено что-то вроде:
      P /путь/Makefile.am
      необходимо выполнить:
      $ ./configure --enable-debug=full --prefix=$KDEDIR --with-kdelibsdoxy-dir=$KDEDIR/share/doc/HTML/en/kdelibs-apidocs
      используя параметры ./configure которые использовались при первой компиляции.


  4. Скомпилируйте и установите kdevelop:
    $ make
    $ make install (под root)
    If for some reason the make command fails you can force it to ignore errors by adding the -k option like this:
    $ make -k
    $ make -k install (as root)
    Если по какой-либо причине команда make завершается неудачно, то можно игнорировать ошибки указывая параметр -k. В зависимости от степени тяжести ошибки можно получить как работающую так и неработающую версию kdevelop. Так что, используйте "make -k install" на собственное усмотрение!!!. В любом случае такие ошибки обычно уходят через пару часов, так что можно попытаться обновиться попозже перед страшной :) командой make install.


Это все! Теперь вы имеете наиболее свежую версию KDevelop. Для запуска выполните:
$ kdevelop

Все, что необходимо делать для поддержки свежей версии - это повторять предыдущие шаги в случае если кто-либо в списке рассылки kdevelop-devel@kdevelop.org исправит ошибку.
Так что приглашаем подписаться на список рассылки.

Ошибки следует сообщать с помощью формы сообщений об ошибках. Не следует сообщать там об ошибках компиляции. Для таких случаев необходимо использовать список рассылки.

Взглянуть на ошибки можно в базе данных об ошибках.
