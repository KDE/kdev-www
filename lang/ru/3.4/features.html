Новая версия имеет все возможности KDevelop 3.3.x плюс:
  • Улучшена поддержка Qt4
    • Возможность указания каталога Qt при создании проекта
    • Автоматический выбор подходящей версии дизайнера интерфейса для файлов .ui
    • Автоматический выбор подходящей версии qmake в проектах qmake
    • Включение файлов в стиле Qt4
    • Новые параметры для переменной окружения QT в конфигурации проекта
    • Обработка ресурсов Qt4 в менеджере проектов QMake
  • Улучшения в QMake Manager
    • Новый анализатор
    • Корректно обрабатывает и выводит сведения о системе сборки Qt4
    • Области видимости
      • Представление областей видимости в виде узлов дерева в панели Обзор
      • Добавление/Удаление областей видимости всех типов (простые, функции и включаемые файлы)
      • Упрощено исключение файлов из области видимости
      • Возможность отключения подпроектов в областях видимости, включая корректное отображение в интерфейсе
    • Работа с файлами
      • Поддержка относительных имн файлов
      • Поддержка путей к файлам с переменными
    • Поддержка подпроектов вне корневого каталога проекта
    • Поддержка всех документированных и нескольких недокументированных функций QMake
      • Поддержка SUBDIRS = somename, somename.subdir = the_real_dir
      • Поддержка SUBDIRS = somefile.pro
    • Новые параметры сборки
      • рекурсивный запуск qmake
      • Dist-Clean
      • Build File
  • Улучшенный отладчик
    • Новое представление ресурсов памяти
    • Автоматическая пересборка проекта при запуске отладчика
    • Виджет просмотра переменных
      • Точная работа при наличии скрытых переменных
      • Новое меню "формат", поддерживающее специальные форматы "binary" и "character" (доступны также через комбинации клавиш)
      • Комбинация клавиш для удаления точек наблюдения
    • Виджет стека
      • Содержание окна разбито на столбцы
      • Кадры стэка получаются инкрементально, что позволяет, например, отлаживать программы с бесконечной рекурсией.
    • Улучшена работа с точками останова
      • Типы точек останова переименованы для ясности
      • Точки наблюдения теперь всегда задаются по адресу, а не по выражению
      • По щелчке на точке наблюдения появляется окно сообщения
      • Условия точки останова корректно загружаются из файла сеанса
      • Вручную добавленные точки останова также отображаются в списке точек останова
      • Адресные точки останова теперь работают
    • В окне вывода GDB можно отключить вывод "внутренних" команд отладчика
    • Реорганизован интерфейс диалога "Присоединить к процессу". Добавлена строка поиска.
    • Исправления
      • Действие Прервать доступно всегда
      • Расположение gdb задатся по полному имени исполняемого файла, а не просто по его каталогу
    • Работа с GDB через специальный межпрограммный интерфейс (MI) для увеличения производительности
  • Улучшенная поддержака Ctags
    • Теперь возможно работать сразу с несколькими открытыми файлами меток, что позволяет осуществлять молниеносные поиски во внешних исходниках
  • Улучшенный интерфейс пользователя
    • Простой режим IDEAl UI теперь установлен по умолчанию
      • Стал быстрее и содержит меньше ошибок
      • Занимает меньше места на экране
      • Видимое пространство может быть разбито на части, что позволяет KDevelop показывать файлы как в Midnight Commander
      • Навигация с клавиатуры аналогичная IntelliJ IDEA (<alt+1>, <alt+2> и т.д.)
      • Улучшена навигация при помощи клавиатуры, переход фокуса
  • Улучшена функция оформления кода
    • Обновлено до AStyle 1.19
    • Поддержка всех параметров AStyle
    • Улучшено окно настроек
    • Глобальные и настройки, привязываемые к проекту
    • Пакетное оформление кода в файлах
  • Поддержка С++
    • Переключение между заголовком и реализацией
      • Работает внутри одиночного файла
      • корректно работает с методами, возвращающими данные, тип которых определяется через typedef
      • Работает с const-функциями
      • Учитывает используемые директивы при поиске функций из пространств имн
    • Persistant Class Store (базы данных для автозавершения кода)
      • Исправлено нахождение папки kdelibs при импорте сведений о классах KDE
      • Импорт классов Qt4
      • Автоматически найднные пути теперь возможно изменять
    • Варианты дополнения кода (code hinting)
      • Панель состояние отображает тип выражения под курсором
    • Синтаксический анализатор С++
      • Доработан лексический анализатор, извлекающий комментарии из кода, добавляющий их в соответствующие пункты и сохраняющий их в AST
      • Исправлена ошибка, проявляющаяся в том, что при обработке файлов c большим числом макрокоманд анализатору требуется огромный объм памяти
    • Модель кода
      • В модели и репозитории кода теперь хранится больше информации, такой как шаблоны и комментарии.
      • Работает с вложенными классами, объявляемыми предварительно в одном файле, и объявляемыми полностью (определяемыми) в другом.
    • Дополнение кода
      • Стало более наджным
      • Большая часть кода была переписана
      • Работает внутри шаблонов классов
      • Работает внутри функций классов, импортированных с использованием директив
      • Может анализировать намного больше выражений и может легко быть расширен для распознавания операторов (благодаря абстрактному классу операторов)
      • Поддерживает шаблоны классов, шаблоны функций, и даже неявное создание экземпляров шаблонов для шаблонов функций
      • Вычисляется тип для каждого варианта дополнения и показывается фактический тип (после обработки шаблонов)
      • В подсказках к вызовам и списке вариантов дополнения показываются соответствующие комментарии
      • В списке вариантов дополнения элементы сортируются по правам доступа, также показывается метка protected/private для каждого элемента
      • Исправлен баг, вызывающий сбой KDevelop при попытке автодополнения класса, являющегося базовым (родительским) самому себе, и ещ несколько небольших багов
    • Новые контекстные меню, вычисляющие выражения под курсором и позволяющие просматривать (перемещаться по коду) код и модель кода, используя все задействованные типы и переменные
    • Новый мастер классов
      • Исправлены ошибки
      • Создание только заголовков
  • Обзор классов и Навигация
    • Добавлена подсветка имн функций в обзор классов и навигатор
    • Навигатор теперь имеет собственную функцию KCompletion, так что возможно писать имена функций без области видимости, и их дополнение будет работать
    • Восстановление состояния Обзора классов после обновления. Теперь окно обзора классов не закрывается после каждой успешной сборки
  • Быстрое открытие
    • Теперь возможно использовать произвольные области видимости в качестве фильтра при поиске классов и функций
    • Если найдено несколько классов с указанным именем, предлагается выбрать один из них для открытия
  • Поддержка Ruby
    • Шаблон приложения с инфраструктурой Rails
    • Синтаксический анализатор распознат классы внутри модулей
    • В качестве оболочки можно использовать irb
    • Исправлены индексы документации, теперь они корректно работают с Rails и QtRuby
    • Запустить WEBrick можно из меню Запуск.
    • Синтаксический анализатор
      • Распознат "self.method"
      • Устранено зацикливание при определнных условиях
  • Поддержка PHP
    • Новый синтаксический анализатор PHP / автозавершение кода
    • Устранены ошибки, приводящие к сбоям
  • Консоль
    • Больше не синхронизируется с каталогом текущего документа
  • Grep (Поиск в файлах)
    • Новые типы файлов (включая типы файлов ruby), которыми можно ограничивать поиск
    • возможность делать образец регулярным выражением
  • Панели вывода
    • Поддержка компиляторов Intel Fortran и PGI Fortran от Portland Group (pgf90/pgf95)
  • Мастер приложений
    • Использует utf8 для создаваемых проектов - больше никаких проблем с кодировкой
  • Интеграция с Valgrind
    • Используются переменные системного окружения, указываемые в параметрах запуска
  • Разное
    • Добавлены кнопка на панели инструментов и пункт меню "Перейти к последней позиции правки"
    • Улучшена поддержка кодировок
      • Кодировка, указываемая в диалоге открытия файла, теперь учитывается
      • Модуль "Открыть с помощью" позволяет выбрать кодировку из всех доступных в диалоге "Открыть как"
      • Выбранная кодировка сохраняется для каждого открытого файла, и используется при следующей загрузке проекта
      • Кодировку по умолчанию можно определить для проекта в Параметрах проекта
    • Диалог выбора модулей вынесен из параметров проекта в меню Настройка и теперь набор модулей привязывается к профайлу.
    • Для определения набора модулей для вновь созданного или импортированного проекта следует использовать механизм профайлов
    • Параметр "Встраивать дизайнера интерфейса" учитывается :)
