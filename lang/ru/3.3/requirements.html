Перед тем как начать компилировать версию KDevelop, необходимо убедиться, что установлены нижеперечисленные программы. При наличии более старых версий этих программ возможны странные ошибки.
Обычно можно определить версию установленной программы вызывая ее с аргументом "--version". Например, версию autoconf можно определить выполнив: 
   autoconf --version

Программа/Библиотека Ссылка
KDE >= www.kde.org
Qt >= 3.2.3 www.trolltech.com/products/qt
g++ >= 2.95.3 (или совместимый) gcc.gnu.org
GNU make www.gnu.org/software/make
perl >= 5.004 www.perl.com
autoconf >= 2.52 www.gnu.org/software/autoconf
automake >= 1.6 www.gnu.org/software/automake
flex >= 2.5.4 www.gnu.org/software/flex
libtool >= 1.4 www.gnu.org/software/libtool/libtool.html
Berkley DB >= 3.0 and <= 4.1 www.sleepycat.com


Эти инструменты необязательны, что означает что KDevelop будет работать без них. Но будет недоставать определенных возможностей таких как генерация документации или поиск по нескольким файлам. Т.е. по возможности необходимо проверить систему на их наличие.

Следует отметить что для использования поддержки определенного языка потребуется соответствующий компилятор этого языка. KDevelop не ограничивает в выборе компилятора, но предоставляет диалоги параметров компилятора для некоторых из них. В настоящее время доступны диалоги для gcc, g++, f77, pghpf (Portland Group High Performance Fortran Compiler), pgf77 (Portland Group Fortran 77 Compiler), dcc (Borland Delphi Compiler) и fpc (Free Pascal Compiler).
