For a more detailed list of changes visit the CVS ChangeLog. It supports KDE >= . Was tagged on 2005-01-05 and released on 2005-01-13. It contains the new features of KDevelop plus the changes:
  • NEW - Features on the KDevelop section on KDE 3.4 Feature Plan
  • NEW - KDevelop Platform:
    • The first release of "KDevelop Platform" - an architecture to build IDE-like applications using common plugins and shell
    • KDevelop IDE and KDevelop Assistant are now two applications built on top of KDevelop Platform
    • Refactored architecture - reworked interfaces and introduced plugin profiles
  • IMPROVED - KDevelop Shell:
    • Shell is modified to use plugin profiles
    • Notification on external changes in files is performed
    • Main window is showed only after initialization
  • IMPROVED - API Documentation:
    • KDevelop program API is now at http://www.kdevelop.org/HEAD/doc/api/html/index.html
  • IMPROVED - Ruby language support:
    • Ruby debugger is implemented
    • Improved ruby parser - more information available in class view
    • Added Qt and KDE application templates
  • FIXED - Small bugs in Perl language support:
  • IMPROVED - Cpp language support:
    • Debugger breakpoints window made easier to use
    • Enable/disable features operate correctly now
    • Fixed problem that disabled breakpoints do not get disabled if starting debugger the first time since loading the project
    • Context menu becomes cleaner
    • Switch header/implementation can now optionally try to match the current function declaration/definition
    • Code completion database can be removed in the configuration dialog
  • IMPROVED - PHP language support: Code completion improvements
  • IMPROVED - Scripting part:
    • New scripting plugin is added
    • KScript based scripting is supported using KScriptInterface
    • KJSEmbed, Shell and Kommander can be used to run a script
  • IMPROVED - Output view:
    • Speed optimizations (approx. 8%)
    • Jump to the error location now works for all cases
  • IMPROVED -Find In Files: Old search results can be stored besides the current search
  • IMPROVED - Code snippets plugin: Snippets can be divided into groups
  • IMPROVED - Application templates:
    • Installation, instantiation and compilation is fixed for following application templates: wxhello, kxt, kscreensaver, kdevpart, kdevpart2, kofficepart, kateplugin
    • KDevelop plugin and language support templates were updated to match changes in the architecture
    • All occurences of wrong variable substitution in templates are fixed
    • CMake application templates added
  • IMPROVED - Difference viewer:
    • Kate part is excluded from a list of difference viewers
    • KDiff3 is now supported
  • IMPROVED - Source formatter: Preview for custom style is implemented
  • IMPROVED - Generic projectmanager:
    • Another generic project manager was started. It's based on the new architecture where project management GUI is abstracted from a specific buildsystem. Buildsystems are handled using plugins ("project importers, builders and editors")
    • Workspace support is available
    • Project builders: make builder
    • Project importers: automake, custom makefiles
  • IMPROVED - QMake project manager:
    • UI is simplifyed
    • Custom qmake variables are supported
    • Support for libtool archives, pkgconf files, console apps and windows dlls is added
  • IMPROVED - Automake project manager: Support for arbitrary make targets and commands in subproject context menu is implemented
  • FIXED - Many bugs in Custom Makefiles project manager
  • IMPROVED - Valgrind support: Valgrind 2.1 and upper are suported now (proper command line options are used)
  • IMPROVED - GUI Designer Integration: KDevelop is compatible with Kexi Formeditor now
  • IMPROVED - KDevelop Designer:
    • Performance improvements and UI fixes
    • Editing source functionality is now available in integrated state
    • Documentation settings dialog is visually improved
  • IMPROVED - KDevelop Assistant and documentation plugin:
    • Fonts and zoom factors are configurable
    • CHM files are displayed with the index frame
  • IMPROVED - Overall improvements:
    • Reduced startup time
    • Property editor library improvements
    • XDG menus are created
    • Source tree is restructured and all unused parts were removed
  • NEW - Kyzis editor (CVS version, www.yzis.org) now works with KDevelop
  • DELETED - closer, copyto, nedit, buglist, doctreeview, knotes, manager, scripting, sourcenav, cvsclient, cervisia, classbrowser, old generic project manager and ctags plugins were removed.
  • FIX - around 110 bugs of the KDevelop 3.1.x release series
